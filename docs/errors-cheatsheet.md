# Шпаргалка по ошибкам T-Invest API (ретраи)

Выжимка из официального каталога кодов ошибок (см. `docs/README.md`; на 2026-06 — 178 кодов: 150 INVALID_ARGUMENT,
11 NOT_FOUND, 6 RESOURCE_EXHAUSTED, 3 INTERNAL, 3 PERMISSION_DENIED, 3 FAILED_PRECONDITION,
по 1 UNIMPLEMENTED/UNAVAILABLE/UNAUTHENTICATED). Здесь — только то, что важно для устойчивости ботов.

## Политика ретраев по типу ошибки

| gRPC-тип | Ретраить? | Как |
|---|---|---|
| `RESOURCE_EXHAUSTED` (80xxx) | **Да** | экспоненциальный бэкофф, начать с паузы ≥1 сек; снизить частоту запросов |
| `INTERNAL` (70xxx) | **Да** | бэкофф + повтор; при стойком повторе — стоп и алерт |
| `UNAVAILABLE` | **Да** | сетевой сбой/недоступность — повтор с бэкоффом |
| `INVALID_ARGUMENT` (30xxx) | **Нет** | баг в запросе — чинить код, повтор бесполезен |
| `NOT_FOUND` (50xxx) | **Нет** | неверный id/инструмент — не повторять |
| `PERMISSION_DENIED` / `UNAUTHENTICATED` | **Нет** | токен/права — повтор не поможет |
| `FAILED_PRECONDITION` (90xxx) | **Нет** | нужно действие (подтверждение/квал) — повтор без изменения бессмыслен |

## Ключевые коды

| Код | Тип | Смысл | Действие |
|---|---|---|---|
| **80002** | RESOURCE_EXHAUSTED | **Превышен лимит запросов/мин** | бэкофф, держать ≤50 req/s суммарно (лимиты — см. `docs/README.md`) |
| **80001** | RESOURCE_EXHAUSTED | Лимит открытых стримов | закрыть лишние соединения |
| **80005** | RESOURCE_EXHAUSTED | Стрим завершён по времени жизни | переподключиться (это норма для стримов) |
| **70001/70002** | INTERNAL | Внутренняя/сетевая ошибка сервиса | повтор с бэкоффом; стойко — в поддержку |
| **30034** | INVALID_ARGUMENT | **Not enough balance** (в т.ч. песочница) | в sandbox — `SandboxPayIn`. ⚠️ Известный баг: может прилетать при закрытии шорта несмотря на достаток средств — `daybot.market_exit` доливает кэш и повторяет (см. `gotchas.md`, память [[daybot]]) |
| **90001** | FAILED_PRECONDITION | Need confirmation | для заявок — флаг `confirm_margin_trade` |
| **90002** | FAILED_PRECONDITION | Только для квал. инвесторов | инструмент недоступен под этим статусом |
| **12001/12002** | UNIMPLEMENTED/UNAVAILABLE | Метод не реализован/устарел | не вызывать |

## Практика для ботов (`lab/`, `daybot/`)

- Слой `lab/api.py` уже делает retry/бэкофф — новые вызовы пускать через него, не плодить голый `requests`.
- На `80002` — **не** жёсткий цикл повторов: увеличивать паузу, группировать запросы.
- `30034` при закрытии позиции в песочнице — не паниковать, это известная грабля (обход уже в `daybot`).
- Полный список параметрических `30xxx` (missing/invalid parameter) — в официальном каталоге ошибок,
  это всегда «чинить запрос», а не ретраить.

> Источник: официальный каталог ошибок T-Invest API (gRPC) и HTTP-маппинг — ссылки в `docs/README.md`.
