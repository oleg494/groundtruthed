# Дизайн: sandbox grid-бот (`scripts/sandbox_grid.py`)

Дата: 2026-06-11. Утверждён в диалоге (выбор пользователя: автономный бот → сеточник → живой процесс → подход C).

## Цель

Поиграться с механикой заявок T-Invest API безопасно: автономный грид-бот на песочнице
(`sandbox-invest-public-api.tinkoff.ru`), виртуальные деньги. Учебно-экспериментальный
инструмент, не продакшен и не боевая торговля.

## Безопасность (жёсткие инварианты)

- `BASE` захардкожен на sandbox-домен. Боевой URL в файле не встречается.
- Токен — только `TINVEST_SANDBOX_KEY` из `.env`. `TINVEST_API_KEY` не читается.
  Нет ключа → выход с инструкцией по выпуску sandbox-токена.
- Sandbox-токен на боевых методах не работает по построению (гарантия брокера).

## Параметры (константы в шапке файла)

| Параметр | Значение | Комментарий |
|---|---|---|
| INSTRUMENT | SBER, uid e6123145-9665-43e0-8413-cd61b8aa9b13 | ликвидная, лот 10 |
| GRID_STEP_PCT | 0.5% | шаг сетки |
| LEVELS | 5 вниз (buy) + 5 вверх (sell) | от цены старта |
| LOTS_PER_LEVEL | 1 лот | |
| POLL_SEC | 5 | ~36 вызовов/мин < лимит 200/мин |
| VIRTUAL_CASH | 100 000 ₽ | SandboxPayIn при первом запуске |

## Жизненный цикл

1. **Старт**: `GetSandboxAccounts` → нет счёта → `OpenSandboxAccount` + `SandboxPayIn`.
2. **Сетка**: цена через `GetLastPrices` → 5 buy-лимиток ниже с шагом 0.5% и 5 sell выше
   (sell только при наличии позиции; иначе уровень «отложен» до появления бумаг).
3. **Цикл** (каждые POLL_SEC): `GetSandboxOrders` + `GetLastPrices` → найти исполненные
   (orderId пропал из активных) → классика грида: buy исполнен → sell на уровень выше;
   sell исполнен → buy на уровень ниже → дозаказать недостающие → state на диск → строка лога.
4. **Ctrl+C**: снять все заявки (`CancelSandboxOrder`), финальный отчёт
   (сделки, оборот, реализованный P&L, длительность).

## Состояние: `analysis/sandbox_grid_state.json`

```json
{
  "account_id": "...", "instrument_uid": "...", "base_price": 322.0,
  "levels": [{"idx": -1, "price": 320.4, "side": "buy", "order_id": "...", "status": "active"}],
  "fills": [{"ts": "...", "side": "buy", "price": 320.4, "lots": 1}],
  "realized_pnl": 0.0
}
```

Перезапуск читает state, сверяет с `GetSandboxOrders` и продолжает. Состояние пишется
атомарно (tmp + rename) после каждого изменения.

## Обработка ошибок

- HTTP 429/5xx — ретраи с backoff (паттерн из `account_full.py`).
- Заявка отклонена (нет денег/бумаг) — уровень помечается `error`, цикл живёт дальше.
- Биржа закрыта — заявки в песочнице принимаются, исполнение по факту котировок;
  бот просто видит отсутствие исполнений.

## Лог

Строка на цикл при изменениях + heartbeat раз в минуту:
`14:32:05 SBER 322.4 | grid 4buy/5sell | pos 10 | fills 3 | P&L +12.4`

## Вне скоупа (YAGNI)

Стопы, трейлинги, несколько инструментов, оптимизация параметров, веб-интерфейс,
стримы (gRPC), уведомления.

## Тест-план

Запуск на живой песочнице: открытие счёта, пополнение, выставление сетки (видно в
`GetSandboxOrders`), 2–3 минуты цикла, Ctrl+C → заявки сняты, отчёт напечатан,
state-файл корректен. Повторный запуск — продолжение без дублей заявок.
