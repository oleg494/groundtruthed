# Daybot — локальный интрадей-бот на фьючерсах (песочница)

Дата: 2026-06-12. Статус: дизайн утверждён в диалоге.

## Цель

Фоновый бот на ПК пользователя (Windows 11): торгует волатильные фьючерсы
внутри дня на ПЕСОЧНИЦЕ T-Invest, ночных позиций не держит. Включается
планировщиком или батником, выключается сам после закрытия сессии.

## Рамки

- Только sandbox API (sandbox-invest-public-api.tinkoff.ru). Боевой токен не используется.
- Лонг-онли: шорты фьючерсов в песочнице закрыты брокером — пробои вниз пропускаются.
- Один новый sandbox-счёт `daybot` (лимит ~10 счетов на токен; ферма занимает 6).
- Инструменты: BR (нефть), NG (газ), GLDRUBF (золото).

## Архитектура

Пакет `daybot/` в tinvest-project, переиспользует `lab/`:

| Модуль | Роль |
|---|---|
| `daybot/run.py` | главный цикл: opening range -> пробои -> ведение стопов/тейков -> закрытие дня |
| `daybot/report.py` | отчёт по дням: сделки, P&L |
| `lab/api.py` | REST-слой sandbox с ретраями (импорт) |
| `lab/instruments.py` | реестр; добавить BR и NG (uid, lot, step, point_rub) |
| `lab/journal.py` | SQLite-журнал; отдельная база `daybot/daybot.db` |

Обвязка Windows: `daybot/start.bat` (pythonw, фон, лог `daybot/daybot.log`),
`daybot/stop.bat` (файл-флаг `daybot.stop` — мягкая остановка),
`daybot/setup_task.bat` (задача планировщика: будни 9:55 МСК).
PID-лок через ctypes OpenProcess (os.kill(pid,0) на Windows не работает).

## Торговая логика (один день)

1. 10:00–10:30 МСК: копить минутные свечи, построить opening range (high/low)
   по каждому инструменту. До 10:30 не торговать.
2. После 10:30, тик 10 с: пробой high диапазона последней ценой, позиции нет,
   входов сегодня < 3 по инструменту -> рыночная покупка на ~30% equity.
   Стоп = low пробойной свечи, но не дальше 0.5% от входа. Тейк = вход + 2x риск.
3. Стоп/тейк ведёт сам бот по last price каждый тик (стоп-заявок в песочнице нет).
4. 23:40 МСК: принудительное закрытие всех позиций, итог дня в журнал, выход.
5. Учёт фьючерсного кэш-бага песочницы: equity с поправкой fut_entry (как в лабе).

## Ошибки и восстановление

- Сетевые сбои: ретраи в lab.api; нет цен > 5 минут при открытой позиции ->
  аварийное закрытие и выход.
- Падение процесса: при старте бот читает портфель; осиротевшие позиции
  закрывает рыночными (диапазон дня утрачен — не угадывать).
- Повторный запуск при живом процессе: PID-лок, второй экземпляр не стартует.
- Выходной/праздник: market_open() == false весь день -> бот пишет событие и выходит.

## Тестирование

- Смоук локально днём при открытой бирже: диапазон строится, вход по пробою
  (или его честное отсутствие), стоп/тейк отрабатывают, stop.bat закрывает позиции.
- Журнал: сделки и equity пишутся в daybot.db, отчёт читается.

## Вне рамок (YAGNI)

Шорты, опционы, перенос позиций, телеграм-уведомления, подбор инструментов
по волатильности дня, реальная торговля.
