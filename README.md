# groundtruthed

Аналитическое пространство вокруг брокерского счёта в Т-Инвестициях (T-Bank): анализ
рынка и портфеля по **боевому read-only** токену плюс форвард-тест торговых стратегий в
**песочнице** (виртуальные деньги, реальные котировки MOEX). Реальные сделки не
совершаются — боевой доступ только на чтение.

> Не приложение, а рабочее пространство: код обслуживает анализ, результаты идут в `analysis/`.

> Неофициальный личный проект. Не аффилирован с Т-Банком/Т-Инвестициями и не одобрен ими;
> «T-Invest API», «Tinkoff», «T-Bank» — товарные знаки соответствующих правообладателей,
> упоминаются здесь только для описания совместимости.

Стержень проекта — **дисциплина оракулов и убийства идей**: 32 скрипта `analysis/*_validate.py`
сверяют каждый нетривиальный расчёт с объективным эталоном (серверное значение, безарбитражное
тождество, кросс-эндпоинт), а не верят зелёному юнит-тесту; стратегии проходят конвейер
идея → бэктест → walk-forward → форвард-тест в песочнице, и большинство **убиты** этим конвейером
до реальных денег — репозиторий фиксирует и удачные модели, и честные отрицательные результаты.

## Слои

| Каталог | Что это | Деньги |
|---------|---------|--------|
| `scripts/` | Скрипты к боевому API (read-only): обзор портфеля, экономика счёта, скринеры | боевой токен, только чтение |
| `backtest/` | Событийный бэктестер (stdlib-only, без numpy/pandas): прогон идей на истории за секунды | без денег, оффлайн |
| `lab/` | Торговый слой песочницы + ферма стратегий (форвард-тест 24/7) | песочница |
| `daybot/` | Локальный интрадей-бот на слое `lab/` (остановлен — см. `CLAUDE.md`, раздел «Песочные боты») | песочница |
| `docs/` | Свои гайды по T-Invest API + ссылки на официальные источники | — |
| `deep/` | Deep-research отчёты: рыночные измерения с оракул-верификацией | — |
| `analysis/` | Артефакты анализа + слой **проверок-оракулов** (`*_validate.py`, см. ниже) | — |

Подробности по каждому слою — в его README (`lab/README.md`, `backtest/README.md`,
`docs/README.md`) и в [`CLAUDE.md`](CLAUDE.md) (правила и устройство для ИИ-ассистента).

## Конвейер стратегии

```
идея → backtest (отсеять мусор за секунды) → lab/daybot (форвард-тест в песочнице:
       выживает ли в живом потоке с задержками и багами брокера) → [только потом] реальные деньги
```

Бэктест — фильтр на входе (быстро, но врёт идеальным исполнением и подгонкой под историю).
Форвард-тест в песочнице — финальная проверка перед реальностью (ловит проскальзывание,
задержки и баги симулятора, которых в истории нет). Любые оптимизированные параметры —
только через walk-forward; стратегия обязана бить бенчмарки buy&hold и random. На момент
публикации конвейер отбраковал 12+ кандидатов стратегий (обе волны botdev, ORB-daybot,
4 стратегии фермы по Deflated Sharpe = 0%) — это не провал проекта, а его смысл: дешёвая
проверка гипотез до того, как они коснутся денег.

## Быстрый старт

```bash
cp .env.example .env          # вписать свои токены и account id
pip install -r requirements.txt

python scripts/explore.py            # проверка боевого подключения + обзор портфеля (read-only)
python -m backtest demo --html demo.html   # демо бэктестера на синтетике
python -m pytest tests/ -q           # тесты бэктестера (~155, 3 xfail = известные баги движка)
python analysis/run_all_validations.py     # прогон проверок-оракулов (--fast — без медленных)

python -m lab.runner                 # ферма стратегий в песочнице (Ctrl+C — graceful stop)
python -m lab.report                 # отчёт фермы
python -m daybot.run                 # daybot (остановлен, см. CLAUDE.md перед запуском)
python -m daybot.report              # отчёт daybot
```

## Проверки-оракулы (`analysis/*_validate.py`)

Слой верификации: 32 скрипта, каждый сверяет данные/расчёты T-Invest с **объективным
оракулом** (серверное число, безарбитражное тождество, кросс-эндпоинт) — где возможно, бит-в-бит.
Это регресс-набор на корректность данных и конвенций: если API поменяет поведение, проверка
покраснеет. Карта — `analysis/VALIDATION_INDEX.md`, раннер — `run_all_validations.py`.

Главная польза — **вскрытые недокументированные конвенции** (теперь считаются офлайн, без догадок):

| что проверено | результат |
|---|---|
| `GetTechAnalysis` (SMA/EMA/RSI/MACD/BB) | бит-в-бит; RSI=сглаживание Уайлдера, BB=дисперсия ÷N, AVG=(o+h+l+c)/4 |
| YTM облигаций vs поле `yield` | эффективная годовая, ACT/365, расчёты T+1 (±1.3 б.п.) |
| Опционы FORTS, теор-цены | premium-style паритет с дисконтом (R²≈1) → из наклона ставка и форвард |
| Агрегация свечей день/неделя/месяц | старший ТФ = точная OHLCV-свёртка младшего; граница дня = MSK |
| Фундаментал-мультипликаторы | P/E, P/S, P/FCF, маржа, дивидендные — согласованы с сырьём |
| Бета / 52-нед. / сетка цен | бета=дневные за год vs IMOEX; high/low=интрадей; цены кратны шагу |
| Целостность/детерминизм данных | low≤OHLC≤high, идемпотентность, независимость от нарезки окна |
| Имена RPC в коде vs proto-контракты | ловит вызовы несуществующих методов (404, тихо проглоченные try/except) |

Неофициальные подводные камни — в `docs/gotchas.md` (сырой REST не кладёт строку `value` в
денежные поля; НКД не линеен; ГО фьючерса ≠ нотионал×dlong; Инвесткопилка торгует через OTC и т.д.).

## Токены, счета и домены

`.env` (не коммитится, см. `.env.example`) хранит:

- `TINVEST_API_KEY` — боевой **read-only**: MCP `t-invest` и `scripts/` (домен `invest-public-api.tinkoff.ru`).
- `TINVEST_SANDBOX_KEY` — песочница: `lab/`, `daybot/`, `scripts/sandbox_grid.py` (домен `sandbox-invest-public-api.tinkoff.ru`).
- `TINVEST_ACCOUNTS` — свои номера брокерских счетов через запятую (используют `analysis/*_validate.py` и часть `scripts/`).

Боевой ключ в песочный код не подставлять и наоборот — sandbox-токен на боевых методах не работает.

## Правила

- **READ-ONLY на боевом счёте**: не размещать заявки, не менять позиции. Только `get_*`/`list_*`/`find_*`.
- Эксперименты с заявками — только в песочнице.
- Цены фьючерсов и облигаций в методах сервиса MarketData приходят в **пунктах**, не в валюте.
  Денежные величины — `{units, nano}`, конвертировать перед расчётами. Подробности и ссылки на
  официальные источники — `docs/README.md`.
- Лимиты API — по сервисам, не общий счётчик (см. `docs/README.md` → официальные источники).
