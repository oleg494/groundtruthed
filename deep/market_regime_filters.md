# Фильтры рыночного режима для MOEX (ADX/Hurst/RV/RUSFAR) без pandas/numpy

## 1. Цель и ограничения

Задача: построить **простые фильтры рыночного режима** для MOEX, которые:

- различают:
  - **trend / range**,
  - **volatility regime**,
  - **liquidity regime**,
  - **rate regime**,
  - **commodity / ruble regime**;
- **считаются в бэктесте без pandas/numpy** — т.е. по спискам чисел, с рекурсивными формулами, без векторизации;
- используются как **фильтры к стратегиям** (включать/выключать или менять параметры).

Ниже — набор конкретных признаков и оценка перечисленных вами индикаторов.

---

## 2. Trend / Range regime

### 2.1. ADX (Average Directional Index)

**Суть.** Ненаправленный индикатор силы тренда. Высокий ADX ⇒ рынок трендовый, низкий ⇒ флэт [1].

**Расчёт (идея, без кода‑фреймворка):**

1. Для каждого бара:
   - `TR` = max(
     `high - low`,
     `abs(high - prev_close)`,
     `abs(low - prev_close)`
   )
   - `+DM` = `high - prev_high`, если это > `prev_low - low` и > 0, иначе 0
   - `-DM` = `prev_low - low`, если это > `high - prev_high` и > 0, иначе 0
2. **Wilder‑сглаживание** (рекурсивное, легко считать в цикле):

   Для периода `N` (обычно 14):

   - первый `TR_N` = сумма `TR` за первые N баров; далее:
     `TR_N = TR_N_prev - TR_N_prev/N + TR`
   - аналогично для `+DM_N`, `-DM_N`.

3. **DI**:

   - `+DI = 100 * (+DM_N / TR_N)`
   - `-DI = 100 * (-DM_N / TR_N)`

4. **DX и ADX**:

   - `DX = 100 * abs(+DI - -DI) / (+DI + -DI)`
   - первый `ADX_N` = среднее DX за N баров; далее:
     `ADX = (prev_ADX * (N-1) + DX) / N`

**Пороги для режима** [1]:

- `ADX < 20` — рынок, как правило, **range / флэт**;
- `20–25` — переходная зона;
- `> 25` — **устойчивый тренд**.

**Оценка для MOEX:**

- Плюсы:
  - хорошо отделяет тренд от шума,
  - не зависит от направления,
  - легко встроить в event‑based бэктест.
- Минусы:
  - есть лаг за счёт сглаживания,
  - чувствителен к гэпам на отдельных бумагах.

**Практический фильтр:**

- **Trend‑стратегия:** торговать только при `ADX > 25`.
- **Mean‑reversion:** торговать только при `ADX < 20`.

### 2.2. Hurst exponent

**Суть.** Оценивает, насколько ряд **трендовый** или **средне‑возвращающийся**.
Интерпретация [3][4]:

- `H > 0.5` — **персистентность / тренд**;
- `H ≈ 0.5` — **близко к случайному блужданию**;
- `H < 0.5` — **анти‑персистентность / mean‑reversion**.

**Простой метод оценки (через лаг‑вариации)** [1][2]:

1. Берём лог‑цены `x_t = ln(S_t)`.
2. Для лагов `τ = 2..max_lag` (20–100):

   - считаем `d_i(τ) = x_i − x_{i−τ}` для `i = τ..N−1`;
   - считаем дисперсию `Var(d(τ))`.
3. Строим точки `(log(τ), log(Var(d(τ))))`.
4. По этим точкам делаем линейную регрессию `y = m * x + b` (обычный least squares).
5. Оценка: `H = m / 2` [2].

**Порог как фильтр:**

- `H > 0.55–0.6` — активен **trend‑following**;
- `H < 0.45–0.4` — активны **mean‑reversion** стратегии;
- `0.45–0.55` — **выключить** обе.

**Плюсы:**

- Одно число даёт «режим»: тренд / рандом / mean‑reversion.
- В исследованиях Hurst хорошо работает как фильтр режимов для тренд‑стратегий [3][4].

**Минусы:**

- Расчёт тяжёлый (много лагов + регрессия), особенно в rolling‑режиме.
- Сильно зависит от выбора `max_lag` и горизонта данных [1][4].

**Вердикт:**
как **основной фильтр** для «лёгкого» бэктеста — **слишком тяжёлый**; разумно использовать:

- либо **эпизодически** (off‑line, заранее размечать периоды трендов/флэтов),
- либо с **маленьким max_lag** (20–30) и не на каждом баре.

### 2.3. Наклон скользящей средней (MA slope)

**Суть.** Простейший признак направления и силы тренда:

- берём `SMA_fast` и `SMA_slow` (например, 5 и 20 баров);
- считаем относительный наклон:
  `slope = (SMA_fast - SMA_slow) / SMA_slow * 100 [%]`.

**Порог:**

- `slope > +0.5–1%` — **восходящий тренд**;
- `slope < −0.5–1%` — **нисходящий тренд**;
- `|slope| < 0.5%` — **флэт**.

**Плюсы:**

- Реализуется двумя рекурсивными суммами, без тяжёлой математики.
- Минимальные вычислительные затраты.
- Легко комбинируется с ADX:
  ADX говорит «есть ли тренд», slope — «куда».

**Минусы:**

- Чувствителен к выбросам, особенно на коротком `SMA_fast`.
- Сам по себе хуже отделяет тренд от шумного range, чем ADX.

**Вердикт:**
**обязательный базовый фильтр направления**, особенно в комбинации с ADX:

- `ADX > 25` и `slope > 0` — торговать только long‑тренды;
- `ADX > 25` и `slope < 0` — только short‑тренды;
- `ADX < 20` и `|slope| < 0.5%` — можно включать mean‑reversion в диапазоне.

## 3. Volatility Regime

### 3.1. Реализованная волатильность (Realized Volatility)

**Суть.** Историческая вола за окно, по лог‑доходностям [1][3]:

- дневные лог‑доходности: `r_t = ln(P_t / P_{t-1})`;
- выборочная дисперсия за окно `n`:
  `σ² = Σ (r_t − mean_r)² / (n − 1)`;
- реализованная вола (годовая):
  `RV = sqrt(σ²) * sqrt(252)`.

**Простые режимы:**

- `RV < 15%` — **низкая волатильность**;
- `15–30%` — **нормальная**;
- `> 30%` — **высокая / стресс**
  (точные пороги калибровать по истории MOEX).

**Плюсы:**

- Элементарно считается в цикле по списку цен.
- Прямо привязан к риску и размеру стопов.
- Может быть как **фильтр** (не торговать при RV>Х), так и **переключатель плеча**.

**Минусы:**

- Задержка из‑за окна, особенно если окно большое (20–30 дней).

**Вердикт:**
**основной индикатор для volatility regime**. Рекомендация:

- trend‑following: включать при `RV` в **среднем** диапазоне, отключать при экстремально высокой;
- mean‑reversion: избегать высоких RV (частые проливы/гэпы).

---

### 3.2. ATR / Цена (альтернатива)

Идея из практики [1]: нормализовать ATR:

- `ATR_norm = ATR(N) / Price`.

Режимы:

- `ATR_norm < 1%` — **спокойный рынок**;
- `1–2%` — нормальный;
- `> 2%` — **высокая вола**.

Считается так же через Wilder‑сглаживание, как TR в ADX, и очень прост в реализации.

## 4. Liquidity Regime

### 4.1. Объём / средний объём

Самый простой прокси ликвидности:

- `vol_ratio = Volume_today / SMA(Volume, N)`, N=20.

Режимы:

- `vol_ratio < 0.7` — **низкая ликвидность**;
- `0.7–1.3` — **нормальная**;
- `> 1.3` — **повышенный интерес** (может быть как приток, так и паника).

**Плюсы:**

- Считается через одну рекурсивную сумму.
- Работает и для отдельных бумаг, и для фьючерса на индекс/валюту.

**Минусы:**

- Не различает «хороший» и «плохой» объём (шум vs информированный).

---

### 4.2. Breadth (ширина рынка)

**Суть.** Насколько движение по индексу поддерживается большинством бумаг (advance/decline line и др.) [2][20].

Простейший вариант:

- `adv` — число растущих акций из вашего универса;
- `dec` — число падающих;
- `net_pct = (adv - dec) / total * 100`.

Режимы:

- `net_pct > +20%` — **risk‑on**, широкая поддержка;
- `net_pct < −20%` — **risk‑off**, распродажа по всему рынку;
- остальное — нейтрально.

**Плюсы:**

- Хорошо фильтрует «ложные» движения индекса, когда растут только 1–2 тяжёлых компонента.
- Полезно именно для MOEX‑индекса и ETF на него.

**Минусы:**

- Требует **широких данных** по акциям (хотя бы top‑50 IMOEX).
- Чуть сложнее реализуется в инфраструктуре бэктеста.

**Вердикт:**
для **индексных и кросс‑секционных** стратегий — очень полезный фильтр risk‑on/off. Для одиночных инструментов можно обойтись объёмом и спредом.

## 5. Rate Regime (процентные ставки в РФ)

### 5.1. RUSFAR

**Что это.** Benchmark коротких ставок денежного рынка РФ на базе сделок РЕПО с CCP на MOEX [3]. Это «реальная» стоимость денег overnight.

Использование как признака:

- сравнивать текущий RUSFAR с историческим средним и с ключевой ставкой ЦБ;
- чем выше RUSFAR, тем **жёстче** денежный режим.

Простейшая классификация (грубая, для запуска, калибровать по истории):

- `RUSFAR < 10%` — **мягкий** режим (easy);
- `10–15%` — **нейтральный**;
- `> 15%` — **жёсткий** (tight).

**Влияние на MOEX:**

- жёсткий режим (высокие ставки) обычно **давит на акции** и поддерживает рубль;
- мягкий — наоборот, **поддержка акций**, давление на рубль.

---

### 5.2. Ключевая ставка ЦБ РФ (Key Rate)

ЦБ публикует current key rate [21]. Можно использовать:

- `level` — сам уровень:
  `> 15%` — агрессивное сдерживание,
  `< 10%` — стимул для риска;
- `delta` — изменение за последние `X` месяцев:

  - `+50 б.п. и более` — **ужесточение**;
  - `−50 б.п. и более` — **смягчение**.

**Фильтр для акций:**

- при **высоких ставках** и/или цикле ужесточения уменьшать долю долгих позиций в акциях и рублевых риск‑активах;
- для коротких облигаций и денежного рынка — наоборот.

---

## 6. Commodity / Ruble Regime

MOEX исторически **сильно зависит от нефти Brent и курса рубля** [22][1]:

- высокая корреляция MOEX–Brent (~0.76) [1];
- рубль чувствителен к нефти и режиму курсовой политики [6][7].

### 6.1. Признаки по нефти и индексу

1. **Корреляция Brent–IMOEX** (rolling, например 60 дней):

   - `corr > 0.6` ⇒ режим «**commodity‑driven**»: рост нефти почти напрямую поддерживает MOEX;
   - `corr < 0.3` ⇒ **другие факторы** (политика, санкции и т.п.).

2. **Изменение нефти за период**:

   - рост Brent > X% за 20–60 дней ⇒ **поддержка экспортеров**, положительно для индекса;
   - падение наоборот.

(Корреляцию и процентное изменение легко считать в цикле по двум спискам цен, без numpy — ровно так же, как для Hurst регрессия.)

---

### 6.2. Курс USD/RUB

Простой признак:

- `change = USDRUB_now / USDRUB_20d_ago - 1`.

Режимы:

- `change > +10%` — **сильное ослабление рубля**;
  - часто стрессовый режим, возможна распродажа в российских активах;
- `change < −10%` — **укрепление рубля**;
  - бывает при высоких ставках и/или дорогой нефти.

Можно строить фильтры:

- отключать агрессивные long‑стратегии на MOEX при резком ослаблении рубля;
- усиливать хеджирование через валюту/золото.

---

### 6.3. Futures Basis (Contango / Backwardation)

**Суть.** Базис = разница между фьючерсом и спотом [18]:

- `basis = (Futures - Spot) / Spot * 100%`.

Режимы:

- **Contango** (`basis > 0`, скажем `> +2%`) — нормальное состояние рынка:
  фьючерс дороже спота, отражает стоимость переноса и ожидания;
- **Backwardation** (`basis < 0`, скажем `< −2%`) — дефицит «товара» сейчас:
  спот дороже фьючерса, характерен для стрессов и дефицитов на товарных рынках [16][18].

**Применение к MOEX:**

- для нефтяных/сырьевых экспортеров и индексов, сильно завязанных на нефть:
  - **стойкая backwardation** по нефти или по соответствующему фьючерсу ⇒
    рынок может быть в «стресс‑товарном» режиме,
    где прибыль экспортеров сегодня высока, но рынок боится будущего;
- для валютных фьючерсов (USDRUB futures vs spot): аномальный базис
  может сигнализировать о внутренних ограничениях/санкционном дисконнекте [23].

## 8. Практические комбинации фильтров для MOEX

Ниже — **конкретные правила**, которые можно реализовать в любом бэктесте по спискам чисел.

### 8.1. Для трендовых стратегий (фьючерс на IMOEX, крупные акции)

**Фильтр тренда:**

- `ADX_14 > 25`
- `MA_slope(5, 20) > 0`  (рост > 0.5–1% между SMA5 и SMA20)
- (опционально) `H_60d > 0.55`.

**Фильтр волатильности:**

- `15% < RV_20d < 35–40%`.

**Фильтр макро:**

- `rate_regime` по (RUSFAR + key rate)/2 не в сильном tightening
  (например, средняя ставка < ~18%).

### 8.2. Для mean‑reversion / intraday‑reversal

**Фильтр режима:**

- `ADX_14 < 20`
- `|MA_slope(5, 20)| < 0.5%`
- (опционально) `H_60d < 0.45`.

**Фильтр волатильности и ликвидности:**

- `RV_20d < 30%` (слишком высокая RV — опасна для mean‑reversion);
- `vol_ratio > 0.7` (избегать тонкого рынка).

### 8.3. Для «рыночного тайминга» (вес в акциях РФ)

- **Rate regime:**
  - при `RUSFAR` и key rate < 10% увеличивать целевой вес в MOEX,
  - при > 15% — снижать.
- **Commodity / Ruble regime:**
  - `corr(Brent, IMOEX, 60d) > 0.6` и Brent в восходящем тренде ⇒ повышенный вес в экспортеров;
  - резкое ослабление рубля (USDRUB +10% за 20 дней) ⇒ уменьшать риск в рублёвых активах.
- **Breadth / risk‑on:**
  - если `net_pct_breadth > 20%` (широкий рост) — разрешать агрессивные long‑входы;
  - если `< −20%` — включать защиту, снижать плечо.

## 9. Что имеет смысл реализовать в первую очередь

С учётом ваших ограничений (без pandas/numpy, фильтры для стратегий), я бы начал с **минимального работающего набора**:

1. **Trend regime**
   - ADX(14)
   - MA‑slope(5, 20)

2. **Volatility regime**
   - Realized volatility (20‑дневная, годовая)

3. **Liquidity**
   - Volume / SMA(Volume, 20)

4. **Rate regime**
   - текущие RUSFAR ON
   - ключевая ставка ЦБ (разметить историю на easy / neutral / tight)

5. **Commodity / Ruble**
   - rolling изменение USDRUB (20–60 дней)
   - базовый фильтр по направлению Brent (через MA‑slope, даже без строгой корреляции)

**Hurst** и **futures basis** — добавить **вторым этапом**, когда базовые фильтры будут встроены и вы уже увидите, где именно нужен более тонкий режимный фильтр.
