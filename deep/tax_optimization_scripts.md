# Налоговая оптимизация: ИИС-3, ЛДВ, FIFO — правила для скриптов анализа операций

Ниже — итоговое прикладное руководство по налоговой оптимизации для российского инвестора и набор правил, которые можно заложить в скрипты анализа операций T‑Invest (через отчёты/InvestAPI), чтобы:

1. считать after‑tax доходность;
2. автоматически находить действия, улучшающие налоговый результат **без увеличения риска** (или с явной пометкой, что риск растёт).

---

## 1. Базовые налоговые правила, которые должен знать скрипт

### 1.1. Ставки и шкала НДФЛ по инвестициям (2026)

Для налоговых резидентов РФ:

- Доходы от операций с ценными бумагами, дивидендов и купонов облагаются:
  - **13%** — на суммарный инвестиционный доход за год до **2,4 млн ₽**;
  - **15%** — на часть годового дохода **свыше 2,4 млн ₽**.[2][3][5][31][48]

Для нерезидентов:

- Как правило, **30%** с инвестиционных доходов (дивиденды, купоны, прибыль от сделок).[31][48]

Скрипт должен:

- Вести **учёт суммарного инвестиционного дохода по каждому брокеру за год**, чтобы:
  - корректно применять порог 2,4 млн ₽ для 13%/15%;
  - оценивать риск доначисления налога ФНС при сумме дохода у нескольких брокеров > 2,4 млн ₽.[48]

## 2. ИИС‑3: логика для скрипта

### 2.1. Что важно знать

ИИС‑3 с 2024 года — основной тип ИИС.[1][2][5][40][42][45]

Ключевые параметры (для счетов, открытых 2024–2026):

- **Минимальный срок владения:** 5 лет.[1][2][5][40]
- **Вычет на взносы:**
  - база до 400 000 ₽/год, возврат 13% (до 52 000 ₽ в год, при стандартной ставке).[1][2][8]
- **Льгота на доход при закрытии:**
  - прибыль при закрытии счёта **до 30 млн ₽** за все годы владения не облагается НДФЛ; сверх 30 млн ₽ — 13–15%.[1][2][40][42]
- **Дивиденды:** всегда облагаются НДФЛ, льгота ИИС‑3 на них **не распространяется**.[1][2][40]
- Разрешены: российские акции, облигации, ОФЗ, паи российских фондов; валюта — только для покупки бумаг.[1][5]

### 2.2. Правила для скрипта

**Идентификация ИИС‑3:**

- поле счёта `account_type == IIS3` (в API/отчётах);
- хранить дату открытия счёта.

**Скрипт должен:**

1. **Считать срок владения ИИС‑3:**
   - `holding_years = floor((today - open_date) / 365)`;
   - помечать счета, где `holding_years >= 5` как **потенциально льготные для закрытия**.

2. **Считать потенциальный ежегодный вычет на взносы:**
   - Сумма пополнений за год (кроме внутриброкерских переводов).
   - База для вычета: `min(годовой взнос, 400_000)`.
   - Размер вычета: `база * 0.13` (при стандартной ставке).[1][2][8]

3. **Считать потенциальную экономию при льготном закрытии:**
   - Рассчитать суммарную прибыль по ИИС‑3 (все продажи и погашения минус покупки и расходы).
   - Освобождаемая прибыль: `min(прибыль, 30_000_000)`.
   - Экономия НДФЛ: `освобождённая прибыль * ставка (13–15%)`.

4. **Флаг «риск потери льгот»:**
   - Если были частичные выводы/досрочные операции, способные привести к потере льгот (зависит от условий брокера/закона), скрипт помечает счёт как **не рекомендованный к закрытию** без ручной проверки.

**Рекомендации скрипта без увеличения риска:**

- Если инвестор **ещё работает** и платит НДФЛ — при незаполненном годовом лимите по взносам предложить докинуть до 400 000 ₽ (если есть свободные деньги) — это **гарантированная доходность 13%** без рыночного риска.
- Для ИИС‑3, где срок ≥ 5 лет и прибыль значима, но инвестор не планирует вывод — просто показывать «отложенную экономию по налогу» при будущем закрытии (это информативное, а не риск‑драйвящее действие).

## 3. ЛДВ (льгота долгосрочного владения)

### 3.1. Базовые правила

Льгота долгосрочного владения (ЛДВ) даёт инвестиционный вычет по НДФЛ для долгого владения ценными бумагами.[1][2][4][5][10][31]

**ЛДВ‑3 (3‑летняя льгота):**

- Срок владения: ≥ 3 полных года.
- Лимит: **3 млн ₽ за каждый полный год владения** по данной бумаге.
- Макс. вычет по бумаге: `3 000 000 × N`, где N — число полных лет.[1][2][4][5][7][8][10][31]
- Условия бумаги:
  - обращается на российской бирже;
  - куплена не ранее 01.01.2014;
  - эмитент — РФ или ЕАЭС.[1]

**ЛДВ‑5 и РИИ:**

- Пятилетняя льгота и льгота инновационного сектора (РИИ) дают суммарный лимит до **50 млн ₽** освобождённой прибыли.[3][6][7][8][31]
- РИИ — список бумаг ведёт Московская биржа, срок владения ≥ 1 года, правила актуальны до 2028 года.[3][31]

**Ограничения:**

- ЛДВ не распространяется:
  - на дивиденды и купоны;
  - на валюту, фьючерсы, опционы;
  - на бумаги, купленные на ИИС (ИИС рассматривается отдельно).[1][31]
- С 2025 года ЛДВ‑3 **не действует** для бумаг, переведённых на брокерский счёт при закрытии ИИС.[1][26]

В Т‑Инвестициях ЛДВ **применяется автоматически** при продаже подходящих бумаг — брокер не удерживает налог, а в налоговом отчёте отражается вычет за долговременное владение.[1][31]

### 3.2. FIFO и ЛДВ

Брокеры считают налоговую базу по ценным бумагам методом **FIFO** (first in – first out): первые купленные бумаги считаются первыми проданными.[23][29][31]

Это важно для ЛДВ:

- если вы покупали одну и ту же акцию «ступеньками», при продаже сначала продаются **самые старые лоты**;
- ЛДВ рассчитывается по каждому лоту в порядке FIFO: часть проданной позиции может получить ЛДВ, часть — нет, если была куплена недавно.[1][14][31]

**Правило для скрипта:**

- Для каждой продажи бумаги:
  - восстановить цепочку покупок этой бумаги по счёту(ам) в хронологическом порядке;
  - разложить продажу по лотам FIFO;
  - посчитать срок владения для каждой части;
  - определить, какой объём попадает под ЛДВ и в каком размере (с учётом лимитов 3 млн ₽ × годы).

## 4. Купоны и дивиденды

### 4.1. Налогообложение

- Купоны и дивиденды для резидента РФ:
  - входят в **инвестиционную базу** и облагаются 13%/15%.[2][3][5][11][31][41][48]
  - налог обычно удерживается брокером при выплате (в Т‑Инвестициях — автоматически).[31][48]
- ЛДВ **не распространяется** на купоны и дивиденды.[1][5][31][48]
- Для купонов по облигациям — с 2021 года общий порядок: все купоны облагаются НДФЛ (бывшие льготы по ОФЗ и отдельным выпускам в общем случае не действуют).[41][43][48]

Скрипт:

- выделяет все операции типа «DIVIDEND», «COUPON», «BOND_COUPON» и соответствующие удержания налога «DIVIDEND_TAX», «BOND_TAX», «BOND_TAX_PROGRESSIVE» из T‑Invest API.[31]
- ведёт годовой учёт:
  - суммарные **грязные** дивиденды/купоны;
  - суммарный удержанный налог;
  - эффективную ставку налогообложения по каждому инструменту и в целом.

Это нужно для расчёта **after‑tax доходности** по дивидендным/купонным стратегиям.

## 5. Валютная переоценка

### 5.1. Общий принцип

Налог по операциям считается **в рублях**, для каждой сделки отдельно.[27][30][31]

- Покупка: стоимость в рублях по курсу ЦБ на дату расчётов сделки.
- Продажа: выручка в рублях по курсу ЦБ на дату расчётов.
- Прибыль/убыток = (рублёвая выручка – рублёвая стоимость покупки), **даже если в валюте был убыток**.[27][30][31]

Пример из практики:[27][31]

- Купили иностранные акции на $10 000 по курсу 60 ₽/$ → 600 000 ₽.
- Продали на $9 000 по курсу 95 ₽/$ → 855 000 ₽.
- В валюте: −$1 000 (убыток), в рублях: +255 000 ₽ (прибыль).
- НДФЛ: 255 000 × 13% = 33 150 ₽.

**Исключения:**

- По ряду суверенных евробондов Минфина РФ валютная переоценка отменена — налог считается исходя из результата в валюте, пересчитанного по курсу на дату продажи.[31][38][39]

### 5.2. Правила для скрипта

1. Для каждой **валютной** бумаги:
   - при загрузке операций брать:
     - дату расчёта;
     - количество;
     - цену в валюте;
     - курс ЦБ на дату;
   - считать рублёвую стоимость покупки и продажи.

2. Для каждой пары «покупка → продажа» (через FIFO):
   - считать рублёвую прибыль/убыток;
   - определять, где инвестор «платит налог за счёт валютной переоценки при фактическом убытке в валюте».

3. Триггер для оптимизации **без увеличения рыночного риска**:

   - Если планируется продажа иностранного актива при сильном росте валюты (или падении рубля), скрипт может подсветить:
     - «Сделка приведёт к налогу на валютную переоценку в размере X ₽ при долларовой/юаневой просадке».
       Это позволяет:
       - либо **отложить** продажу;
       - либо **реализовать убыток** по другим бумагам в том же году для сальдирования (см. перенос убытков).

## 6. Перенос убытков и сальдирование

### 6.1. Перенос убытков

По НК РФ:[4][19][21][22][24][27]

- Убыток по операциям с ценными бумагами и обращающимися ПФИ можно переносить на последующие годы **до 10 лет**.
- Убытки учитываются **отдельно** по типу:
  - база по ЦБ (акции, облигации, фонды);
  - база по ПФИ (фьючерсы, опционы и др.).
- Нельзя переносить убытки по бумагам и ПФИ, **не обращающимся** на организованном рынке.
- Перенос осуществляется по очередности — сначала более старые убытки.

### 6.2. Сальдирование

В пределах одного брокера за год автоматически сальдируются:[22][23][24][27]

- прибыли и убытки по операциям с ценными бумагами и ПФИ, обращающимися на ОРЦБ;
- по результату получается одна налоговая база по каждому виду.

Чтобы использовать **убытки прошлых лет** у одного или нескольких брокеров, инвестору нужно подать 3‑НДФЛ и приложения с убытками; брокер этого не делает автоматически.[22][23][27]

### 6.3. Правила для скрипта

- Вести **историю годовых финансовых результатов** по каждому типу операций (ЦБ / ПФИ).
- Хранить **неиспользованные убытки** по годам (до 10 лет).
- Для каждого нового года:

  1. Посчитать **прибыль по ЦБ** и **прибыль по ПФИ** за год до вычетов.
  2. Применить убытки прошлых лет, начиная с самых старых, пока:
     - прибыль > 0;
     - есть непросиженные убытки в той же базе.
  3. Оценить экономию налога: `использованный убыток × 13% (или 15%)`.

- В отчёте для пользователя:
  - показать, **сколько убытков можно заявить** через 3‑НДФЛ;
  - оценить, **сколько налога это сэкономит**.

Это не увеличивает рыночный риск: речь только о грамотном учёте уже случившихся результатов.

## 7. Фонды (БПИФ, ПИФ) vs отдельные бумаги

### 7.1. Налоговые особенности фондов

Для БПИФ и ETF на российской бирже:[20][21][24][25][29]

- Дивиденды и купоны, которые **поступают внутрь фонда**, на уровне фонда **налогом не облагаются** — реинвестируются **брутто**.[22][23][25]
- Инвестор платит НДФЛ **только при продаже пая** (или при получении дивидендов по распределяющим БПИФ).[21][25][29]
- К паю БПИФ **применимы ЛДВ‑3 и ЛДВ‑5**, как к обычной ценной бумаге (при выполнении общих условий). Это позволяет ещё и обнулить налог с прироста пая при длинном сроке владения.[1][3][5][29]

Для отдельных бумаг:

- Дивиденды и купоны **облагаются каждый раз при выплате**.[3][11][31][48]
- ЛДВ можно применить только к **приросту стоимости** конкретной акции/облигации, но не к выплате купонов/дивидендов.

### 7.2. Вывод для скрипта

- Для стратегий «долгий пассивный доход» БПИФ часто дают **налоговое преимущество**:
  - отложенный налог на дивиденды/купоны (они растят стоимость пая до продажи без текущего налога);
  - дополнительный выигрыш, если инвестор держит паи ≥ 3 лет и применяет ЛДВ при продаже.

Скрипт может:

1. Находить портфель с **высокой долей дивидендных акций/облигаций**, удерживаемых длительно.
2. Показывать, какова:
   - **эффективная ставка налога** на поток дивидендов/купонных выплат;
   - **гипотетический налог**, если бы аналогичная экспозиция была через БПИФ (налог только при продаже + ЛДВ).
3. Предлагать **неувеличивающие риск варианты**:
   - «Эквивалентный по составу БПИФ X позволил бы снизить текущие налоги на дивиденды/купоны и перенести их на момент продажи пая; оценочная экономия за N лет — Y ₽».

## 8. T‑Invest: данные и структура для скриптов

### 8.1. Какие данные доступны из T‑Invest API

Через `OperationsService` T‑Invest API можно получать:[31]

- список операций (`GetOperations`, `GetOperationsByCursor`);
- поля по каждой операции:
  `id, date, currency, payment, price, quantity, figi, instrument_type, type, operation_type, state, tax, accrued_int, trades...`
- типы операций включают:
  - `OPERATION_TYPE_DIVIDEND`, `OPERATION_TYPE_DIVIDEND_TAX`,
  - `OPERATION_TYPE_BOND_COUPON`, `OPERATION_TYPE_BOND_TAX`,
  - `OPERATION_TYPE_TAX`, `OPERATION_TYPE_TAX_CORRECTION`,
  - прогрессивные варианты: `*_TAX_PROGRESSIVE`.[31]
- отдельный отчёт по иностранным дивидендам (`GetDividendsForeignIssuerReport`):
  `dividend_gross, tax, dividend_amount, external_commission, currency, payment_date, record_date, isin, issuer_country...`.[31]

В налоговом отчёте Т‑Инвестиций (PDF/Excel) содержатся:[31]

- итоговые расчёты по налогам;
- детализация операций купли‑продажи;
- дивиденды и купоны с удержанными налогами;
- вычеты за ЛДВ;
- суммы налога, подлежащего удержанию и фактически удержанного.

### 8.2. Общий алгоритм скрипта

1. **Выгрузка всех операций** за нужный период по всем счётам.
2. **Классификация операций**:
   - покупки/продажи;
   - дивиденды, купоны;
   - удержания налога;
   - поступления/выводы денежных средств.
3. **Реконструкция лотов по FIFO** для каждой бумаги и счёта.
4. **Расчёт:**
   - реализованной прибыли/убытка по каждой продаже;
   - потенциальной ЛДВ (по каждому лоту и сделке);
   - валютной переоценки для иностранных активов;
   - годовых результатов по базам (ЦБ/ПФИ);
   - фактически уплаченных налогов (по типам дохода).
5. **Расчёт after‑tax доходности портфеля**:
   - учитывая:
     - уже уплаченные налоги (дивиденды/купоны/продажи);
     - отложенные налоги от нереализованной прибыли;
     - потенциальный эффект ЛДВ и переносов убытков.
6. **Поиск оптимизационных действий** (см. далее).

## 9. Автоматический поиск действий, улучшающих налоговый результат

Скрипт должен не только считать, но и генерировать **список рекомендаций**, разделяя:

- действия **без увеличения рыночного риска**;
- действия, при которых риск меняется (помечать явно).

### 9.1. Действия без увеличения рыночного риска

1. **Максимизация вычета на ИИС‑3:**

   - Если:
     - у инвестора есть ИИС‑3;
     - годовой взнос < 400 000 ₽;
     - есть облагаемый НДФЛ доход (зарплата и др.).
   - Рекомендация: «Пополните ИИС‑3 ещё на X ₽ до конца года → ожидаемый возврат НДФЛ Y ₽».

2. **Перенос убытков прошлых лет:**

   - Если:
     - в истории обнаружены перенесённые убытки;
     - в текущем году есть прибыль по соответствующей базе.
   - Рекомендация: «Подайте 3‑НДФЛ и заявите убыток прошлых лет Z ₽ → экономия налога ~Z × 13–15%».

3. **Корректный учёт ЛДВ уже реализованных сделок:**

   - Если сделка подпадает под ЛДВ по условиям, но брокер удержал налог (бывает при сложных кейсах или смене брокера).
   - Рекомендация: «Укажите ЛДВ по сделке X в декларации → вернуть ~N ₽».

4. **Уточнение статуса прогрессивной ставки:**

   - Если совокупный доход у всех брокеров > 2,4 млн ₽, а у части удержан налог 13%.
   - Рекомендация: «ФНС может доначислить 2% с суммы превышения P ₽; учитывать при планировании вывода средств».

### 9.2. Действия с потенциальным изменением риска (помечать отдельно)

1. **Не продавать позиции за несколько месяцев до наступления ЛДВ:**

   - Если:
     - бумага удерживается 2.5–2.9 года;
     - значительная нереализованная прибыль.
   - Рекомендация: «До ЛДВ‑3 по бумаге X осталось N дней. Продажа сейчас приведёт к налогу ~Y ₽, при ожидании — налог можно снизить/обнулить. Риск: рыночная цена может измениться».

2. **Tax‑loss harvesting (фиксация убытков до конца года):**

   - Если:
     - по некоторым бумагам – существенный нереализованный убыток;
     - в этом же году есть большая реализованная прибыль.
   - Рекомендация: «Рассмотрите продажу убыточных бумаг A, B, C для сальдирования прибыли ~P ₽ → экономия налога ~P × 13–15%. Риск: возможна упущенная выгода при восстановлении цены».

3. **Замена отдельной дивидендной экспозиции на БПИФ:**

   - Если:
     - долгосрочная стратегия, высокая дивидендная доходность;
     - есть аналогичный по составу БПИФ.
   - Рекомендация: «Перевод экспозиции по сектору X в БПИФ Y уменьшит текущие выплаты НДФЛ и перенесёт налог на момент продажи пая (с возможным применением ЛДВ). Риск: отличия трека фонда от индивидуального портфеля».

---

## 10. Как считать after‑tax доходность

Итого, для **after‑tax** расчёта скрипт должен:

1. Для выбранного периода собрать:

   - начальную рыночную стоимость портфеля;
   - конечную рыночную стоимость;
   - все денежные потоки:
     - пополнения/выводы;
     - дивиденды/купоны (gross и net);
     - уплаченные налоги (по типам операций).

2. Разделить:

   - **реализованный** результат (по сделкам за период, после налогов);
   - **нереализованный** результат (по текущим позициям),
   - оценить отложенный налог:
     - без ЛДВ;
     - с гипотетическим применением ЛДВ.

3. Рассчитать:

   - доходность **до налогов**;
   - доходность **после налогов**;
   - годовую after‑tax доходность;
   - **tax drag** — снижение доходности из‑за налогов.

4. В отчёте показать:

   - вклад каждого фактора:
     - купоны/дивиденды;
     - реализованные сделки;
     - валютная переоценка;
     - льготы (ИИС‑3, ЛДВ, перенос убытков);
     - налоги по ставкам 13% и 15%.
