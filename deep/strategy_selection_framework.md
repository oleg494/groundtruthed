# Фреймворк отбора стратегий перед sandbox-forward-test

Ниже — целостный, **минимально необходимый, но строгий** фреймворк отбора стратегий для частного инвестора на MOEX **перед запуском sandbox‑forward‑test**. Все метрики и пороги — ориентировочные, но достаточно консервативные для частного капитала.

Структура:

1. Общие принципы и объем данных
2. Фильтры ликвидности (liquidity filters)
3. Walk‑forward + anchored / rolling OOS
4. Deflated Sharpe + bootstrap
5. Стабильность параметров (parameter stability)
6. Чувствительность к издержкам (cost sensitivity)
7. Сравнение с buy‑and‑hold и с random
8. Разделение по режимам рынка (regime split)
9. Итоговое правило pass/fail перед sandbox‑forward‑test

---

## 1. Общие принципы и объем данных

**Минимальные требования к историческим данным:**

- Длительность бэктеста:
  - Акции / фьючерсы на индекс: **≥ 5 лет**
  - Высокочастотные / внутридневные системы: можно **3–4 года**, но с очень большим числом сделок
- Минимальное число сделок:
  - **жёсткий минимум**: ≥ 100 сделок
  - **рекомендуемый минимум**: ≥ 200–300 сделок
  (для внутридневных систем — стремиться к ≥ 500–1000 сделок)

**Обязательные анти‑артефактные требования:**

- Нет look‑ahead (используются только данные, доступные на момент принятия решения)
- Учтены все комиссии и реалистичный спред
- Включены делистинги / корпоративные события (по возможности)

---

## 2. Фильтры ликвидности (Liquidity Filters)

Применяются к каждому инструменту, по которому стратегия торгует.

**Для акций MOEX:**

| Критерий | PASS | FAIL |
|---|---|---|
| Средний дневной оборот за последний год | ≥ 50 млн RUB для «основных» бумаг; ≥ 10 млн RUB для более нишевых | < 10 млн RUB |
| Средний дневной объем (штук) | ≥ 10 000 акций/день | < 5 000 акций/день |
| Средний относительный спред (по закрытию и/или внутридня) | спред ≤ 0,15 % | спред > 0,25 % |
| Доля предполагаемого объема сделки от среднедневного | средний размер позиции по бумаге ≤ 5 % от среднего дневного оборота | > 10 % |
| Торговые дни | бумага торгуется ≥ 90 % календарных торговых дней за период | < 85 % |

**Жёсткое правило:**
Если хотя бы один из **жёстких** ликвидностных критериев (оборот, спред, доступность торгов) не выполняется — **стратегия FAIL до любых остальных тестов**.

## 3. Walk‑forward + Anchored / Rolling OOS

### 3.1. Базовая схема Walk‑Forward

- Делим историю на блоки:
  Пример: 60 % IS (in‑sample) / 40 % OOS (out‑of‑sample) или 50/50.
- Запускаем **минимум 5 полных WF‑циклов**.
  Каждый цикл: оптимизация на IS → тест на следующем OOS‑окне.

**Пороговые значения по результатам WF:**

- OOS Sharpe Ratio (по всем OOS‑периодам совокупно):
  - **PASS**: Sharpe_OOS ≥ 0,7
  - **FAIL**: Sharpe_OOS < 0,5
- Деградация OOS vs IS:
  - Пусть R_IS — годовая доходность в IS, R_OOS — в OOS.
  - **PASS**: R_OOS ≥ 0,65·R_IS (деградация ≤ 35 %)
  - **FAIL**: R_OOS ≤ 0,5·R_IS (деградация ≥ 50 %)
- Доля прибыльных WF‑окон (по итоговой доходности OOS‑блока):
  - **PASS**: ≥ 60 % окон положительные
  - **FAIL**: < 50 %
- Максимальная продолжительность непрерывной просадки в OOS:
  - **PASS**: ≤ 6 месяцев
  - **FAIL**: > 12 месяцев

### 3.2. Anchored OOS

- Берём первый отрезок данных как IS (например, первые 3 года),
  затем последовательно **расширяем** тестируемый OOS (anchored walk‑forward).

**Пороги:**

- Sharpe по всей anchored‑OOS выборке:
  - **PASS**: Sharpe_anchored_OOS ≥ 0,65
  - **FAIL**: < 0,5
- Calmar (R / MaxDD) по anchored‑OOS:
  - **PASS**: Calmar ≥ 0,8
  - **FAIL**: < 0,5

### 3.3. Rolling OOS

- Скользящее окно одинаковой длины (например, 2 года IS → 6–12 мес OOS).
- Для каждого окна: пере‑оптимизация параметров → тест на новом OOS.

**Пороги:**

- Средний Sharpe по всем rolling‑OOS:
  - **PASS**: средний Sharpe ≥ 0,7
  - **FAIL**: < 0,5
- Стандартное отклонение Sharpe между окнами:
  - **PASS**: σ(Sharpe_OOS) ≤ 0,4
  - **FAIL**: > 0,6
- Доля прибыльных rolling‑OOS окон:
  - **PASS**: ≥ 55 %
  - **FAIL**: < 50 %

## 4. Deflated Sharpe + Bootstrap

### 4.1. Deflated Sharpe Ratio (DSR)

Используется формула Bailey & López de Prado (учёт:
длины ряда, не‑нормальности, количества протестированных стратегий/параметров и пр.) [1][2].

**Практические пороги:**

- DSR (для годовых доходностей OOS или всей OOS‑выборки):
  - **PASS**: DSR ≥ 0,5 (на уровне доверия 95 %)
  - **FAIL**: DSR < 0,3
- Отношение DSR к «сырым» Sharpe:
  - Пусть SR_raw — классический Sharpe, DSR — скорректированный:
  - **PASS**: DSR ≥ 0,5·SR_raw
  - **FAIL**: DSR ≥ 0,85·SR_raw при очень большом SR_raw
    (сигнал возможного multiple‑testing/selection bias)

**Обязательное правило:**
Стратегия должна вести учёт **количества протестированных вариантов** (N стратегий / параметров). Если N неизвестно или занижено — считать DSR недействительным и **стратегию FAIL**.

### 4.2. Bootstrap (блок‑bootstrap по доходностям)

Метод:

- Используем **block bootstrap** (размер блока ≈ средний срок удержания позиции или 5–10 баров для внутридня).
- Количество resample‑серий: **≥ 1000**.

Проверяем 95 % доверительные интервалы (CI):

| Метрика | PASS | FAIL |
|---|---|---|
| Sharpe Ratio | нижняя граница CI_95(SR) > 0,3 | нижняя граница ≤ 0 |
| Годовая доходность | нижняя граница CI_95(R_annual) > 0 % | нижняя граница ≤ −5 % |
| Profit Factor | нижняя граница CI_95(PF) > 1,1 | нижняя граница ≤ 1,0 |
| Максимальная просадка (MaxDD_obs — наблюдаемая MDD) | верхняя граница CI_95(MaxDD) ≤ 1,5·MaxDD_obs | ≥ 2·MaxDD_obs |

## 5. Стабильность параметров (Parameter Stability)

Тестируется область вокруг оптимальных параметров.

### 5.1. Локальные возмущения (±10 % и ±20 %)

Для каждого ключевого параметра (период MA, пороги сигналов и т.п.):

- Строим сетку: оптимум × (0,8; 0,9; 1,0; 1,1; 1,2) и т.п.
- Прогоняем стратегию **без повторной оптимизации**.

**Метрики и пороги:**

- Средняя относительная просадка доходности при ±10 %:
  - Пусть R* — доходность в оптимуме, R_±10 — доходность при возмущении.
  - **PASS**: |R_±10 − R*| / R* ≤ 25 %
  - **FAIL**: > 40 %
- Доля параметрических точек в ±20 %, остающихся прибыльными:
  - **PASS**: ≥ 80 % комбинаций в «кольце» ±20 % дают R > 0
  - **FAIL**: < 60 %
- Положение оптимума:
  - **PASS**: максимум Sharpe/доходности находится **не вблизи границы** допустимого диапазона (не ближе 5 % к краю)
  - **FAIL**: оптимум на краю параметрического диапазона
- Качественно: карта параметров (heatmap) должна показывать «плато», а не острый «шип».
  Если максимум — узкий пик, где небольшое сдвижение резко рушит метрики → **FAIL**.

## 6. Чувствительность к издержкам (Cost Sensitivity)

Реалистичные издержки для частного инвестора на MOEX (порядок оценок):

- Комиссии брокера + биржи: **~0,05–0,1 % за сторону** (0,1–0,2 % round‑trip)
- Спред и проскальзывание: **0,05–0,15 % round‑trip** на ликвидных бумагах
- Консервативное базовое допущение: **0,2 % на сделку** (round‑trip),
  стресс‑сценарии: 0,3 % и 0,4 %.

**Тестирование:**

Прогнать стратегию минимум в трёх режимах:

1. Базовые издержки: 0,2 % / сделку
2. Повышенные: 0,3 %
3. Стресс: 0,4 %

**Пороги:**

| Уровень издержек | PASS | FAIL |
|---|---|---|
| Базовый (0,2 %) | Sharpe ≥ 0,8, годовая доходность > 0 | Sharpe < 0,5 или годовая доходность ≤ 0 |
| Повышенный (0,3 %) | Sharpe ≥ 0,5, годовая доходность ≥ 0 | Sharpe < 0,3 или годовая доходность < 0 |
| Стресс (0,4 %) | совокупная доходность за весь период ≥ 0 | накопленный результат < 0 |

Дополнительно:

- Оценить **точку безубыточности по издержкам**:
  - Минимальный уровень комиссии c*, при котором R(c*) = 0.
  - **Желательно**: c* ≥ 0,35–0,4 %
    Если уже при 0,25–0,3 % стратегия умирает → **слишком хрупкая**.

## 7. Сравнение с Buy‑and‑Hold и Random

### 7.1. Сравнение с Buy‑and‑Hold

Бенчмарк:

- Для акций — индекс MOEX (или соответствующий секторальный индекс),
  либо простое buy‑and‑hold в торгуемых бумагах аналогичной риск‑экспозиции.

**Пороги:**

| Метрика | PASS | FAIL |
|---|---|---|
| Годовая **избыточная доходность (α)** | α ≥ 3 % годовых | α ≤ 0 % |
| Информационный коэффициент (IR = α / Tracking Error) | IR ≥ 0,4 | IR < 0,2 |
| Up‑capture (в росте рынка) | ≥ 85 % от доходности индекса на растущих участках | < 70 % |
| Down‑capture (в падении рынка) | ≤ 80 % (желательно ≤ 70 %) | ≥ 95 % |
| Доля периодов (месяцев), в которых стратегия обгоняет бенчмарк | ≥ 55 % | < 45 % |

### 7.2. Сравнение с Random

Идея: показать, что стратегия статистически значимо превосходит «случайные» правила с аналогичной частотой сделок и распределением по времени.

Подход:

- Генерируем минимум **1000** случайных стратегий:
  - Случайное время входа/выхода при сохранении структуры частоты сделок и средних удержаний
  или
  - Случайные перестановки (+/−) сделок/сигналов
- Считаем распределение Sharpe и доходностей по «случайному ансамблю».

**Пороги:**

| Метрика | PASS | FAIL |
|---|---|---|
| Sharpe стратегии vs перцентили Sharpe случайных | SR_strategy > SR_random,95 | SR_strategy < SR_random,75 |
| Доходность стратегии vs перцентили случайных | R_strategy > R_random,90 | R_strategy < R_random,80 |
| p‑value (вероятность получить не хуже результат случайной стратегией) | p < 0,05 | p > 0,15 |
| Ожидаемая доля случайных стратегий, обгоняющих вашу | < 10 % | > 25 % |

## 8. Разделение по режимам рынка (Regime Split)

Режимы можно определить по индексу MOEX:

- **Bull**: индекс > 200‑дневной SMA **и** SMA наклон вверх
- **Bear**: индекс < 200‑дневной SMA **и** SMA наклон вниз
- **Sideways**: всё остальное (боковик / нейтральный режим)

Считаем метрики по каждому режиму отдельно.

**Пороги:**

- Bull‑режим:
  - **PASS**: Sharpe_bull ≥ 0,6
  - **FAIL**: < 0,3
- Bear‑режим:
  - **PASS**: Sharpe_bear ≥ −0,3 (т.е. умеренный минус)
    или лучше, если стратегия защитная.
  - **FAIL**: Sharpe_bear < −0,8 (слишком больно в падениях)
- Sideways‑режим:
  - **PASS**: Sharpe_side ≥ 0,2
  - **FAIL**: < −0,2
- Разрыв между лучшим и худшим режимом:
  - **PASS**: |SR_best − SR_worst| ≤ 1,0
  - **FAIL**: > 1,8 (слишком различное поведение)
- Доля лет (или крупных отрезков), в каждом режимном типе с положительной доходностью:
  - **PASS**: хотя бы 50 % лет в каждом присутствовавшем режиме прибыльны
  - **FAIL**: < 30 % прибыльных лет в каком‑либо режиме без внятной логики стратегий/фильтров под этот режим

## 9. Итоговое правило Pass/Fail перед Sandbox‑Forward‑Test

Для простоты можно ввести уровни приоритета:

- **P0 (жёстко обязательные тесты):**
  1. Ликвидность (liquidity filters)
  2. Объём данных и число сделок
  3. Walk‑forward (включая OOS‑показатели)
  4. Deflated Sharpe Ratio
  5. Чувствительность к издержкам (cost ×2)

- **P1 (очень желательные, но при идеальном P0 допустимы небольшие отступления):**
  6. Bootstrap CI
  7. Стабильность параметров
  8. Regime split (нет катастроф в одном из режимов)

- **P2 (усиливающие уверенность, но не всегда критичные для очень простых стратегий):**
  9. Сравнение vs buy‑and‑hold
  10. Сравнение vs random

### Минимальный строгий критерий допуска в sandbox‑forward‑test

**Стратегия допускается в sandbox‑forward‑test, только если:**

1. Все **P0‑критерии строго PASS**:
   - Ликвидность всех реальных торгуемых инструментов соответствует порогам
   - Длина истории и число сделок ≥ указанных минимумов
   - WF‑OOS Sharpe ≥ 0,7, деградация OOS ≤ 35 %
   - DSR ≥ 0,5 (95 % CL), учтено корректное число тестов
   - При удвоении издержек стратегия остаётся неубыточной за весь период
2. Из **P1**:
   - Bootstrap: нижний 95 %‑квант Sharpe > 0,3 и R_annual > 0
   - Параметрическая устойчивость: ≥ 80 % прибыльных точек в ±20 %, без острого «шила»
   - В bearish‑режиме Sharpe ≥ −0,3 (нет смертельных просадок)
3. Из **P2** желательно:
   - α vs MOEX ≥ 3 % годовых и IR ≥ 0,4
   - p‑value vs random < 0,05

Если стратегия проваливает **любой P0‑тест** → **НЕ ДОПУСКАТЬ** к sandbox‑forward‑test, независимо от красоты остальной статистики.
