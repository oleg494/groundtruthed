# D. Bug-hunt челлендж — результат

Проба чистой способности к code-review под оракулом из известного списка багов.

## Постановка
В самостоятельную копию реального модуля `backtest/metrics.py` (чистые функции —
по каждому багу однозначная математическая истина) внесено **6 тонких багов** +
**1 «красная селёдка»** (подозрительное, но корректное место — проверка на ложные
срабатывания). Копия положена ВНЕ репозитория (`/tmp/bughunt/portfolio_metrics.py`),
без git и без оригинала рядом — чтобы исключить сравнение-диффом. Свежий subagent
(general-purpose) получил только этот файл и задачу «найди ошибки корректности»;
число багов ему НЕ сообщалось.

## Внесённые баги (оракул)
| # | Место | Баг | Severity |
|---|---|---|---|
| 1 | `_max_drawdown` | `dd > mdd` вместо `< mdd` → просадка всегда 0 | high |
| 2 | `compute_metrics` | `years = span_days/252` вместо `/365.25` (CAGR на торг. днях) | high |
| 3 | `compute_metrics` | Sortino: делитель `nneg` вместо `len(rets)` (все наблюдения) | medium |
| 4 | `compute_metrics` | `profit_factor = gross_loss/gross_win` — перевёрнут | high |
| 5 | `compute_metrics` | `avg_loss = +gross_loss/len` — без минуса (должен быть <0) | medium |
| 6 | `compute_metrics` | `best, worst = min, max` — перепутаны местами | medium |
| — | `_median_dt` | чётная длина → верхний элемент медианы | НЕ баг (селёдка) |

## Результат
- **Найдено: 6 / 6.** Точность 100% (ноль ложных срабатываний), полнота 100%.
- На «красную селёдку» агент **не клюнул** — не пометил `_median_dt` как баг.
- Все severity проставлены верно; confidence high по всем находкам.

## Вывод
Под оракулом с заранее известным списком LLM-in-harness отработал code-review
без подсказок на отлично: и точность, и полнота = 100%. В отличие от поиска
торгового эджа (где оракула не было и результат был только отрицательный), здесь
результат объективно измерим — и он положительный.
