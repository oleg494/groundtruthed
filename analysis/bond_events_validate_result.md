# Кросс-эндпоинт: GetBondEvents == GetBondCoupons

**Дата:** 2026-06-19 · `analysis/bond_events_validate.py` · READ-ONLY · ОФЗ 26238/26240/26248

## Оракул и результат
Два независимых метода отдают купонный график: `GetBondCoupons` и `GetBondEvents`
(type=EVENT_TYPE_CPN). Оракул — их взаимное совпадение по датам и суммам выплат.

**102/102 купонов совпали** по сумме, ни одной даты только в одном методе:
| облигация | coupons | events | общих | расхождений |
|---|---|---|---|---|
| ОФЗ 26238 | 40 | 40 | 40 | 0 |
| ОФЗ 26240 | 30 | 30 | 30 | 0 |
| ОФЗ 26248 | 32 | 32 | 32 | 0 |

## Вывод
Купонный график облигации согласован между двумя эндпоинтами — можно брать из любого. `GetBondEvents`
дополнительно отдаёт другие типы событий (оферты CALL, погашение MTY, конвертации CONV), а
`GetBondCoupons` — детали купонного периода (start/end/period, использованные в `bond_ytm_validate` и
проверке НКД). Данные не противоречат друг другу. Дополняет `bond_ytm_validate` (YTM из этого же
графика сошлась с полем yield).
