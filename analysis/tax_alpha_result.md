# Налоговая альфа по реальному счёту — результат сканирования

Дата прогона: 2026-07-02. Скрипт: `scripts/tax_alpha_scan.py` (read-only, боевой токен,
только `Get*`; ничего не исполняет — все рекомендации к РУЧНЫМ действиям).

> Конкретные суммы, даты, лоты и тикеры этого прогона относятся к реальному счёту автора и
> не публикуются — здесь только методология, проверенные соотношения и найденные баги.

## Метод

1. `GetAccounts` → все открытые счета; по каждому `GetOperationsByCursor` с даты открытия
   (рекурсивная нарезка окна — обход бага курсора «nextCursor отдаёт ту же страницу»;
   БЕЗ фильтра `state` — фильтр Executed теряет операции; дедуп по `id`).
2. Знаковый FIFO по ISIN (паттерн `analysis/tax_reconcile.py`) → открытые лоты
   с датой/ценой покупки + закрытия по годам.
3. Цены `GetLastPrices` (облигации — `price/100*nominal`, без НКД), P&L и срок per-лот,
   ЛДВ-даты, происхождение лота (переводы `INPUT_SECURITIES` метятся «ИИС неизвестно»).
4. Реализованная база 2026 + дивиденды/купоны + удержанные `TAX*`.
5. Альфа-поиск: harvesting (wash-sale в РФ НЕТ), ЛДВ-рубежи, корзины ст. 214.1 НК.

## Сверка с оракулами (дисциплина проекта)

| Проверка | Эталон | Результат |
|---|---|---|
| Открытые FIFO-лоты | `GetPositions.securities.balance` | **PASS бит-в-бит** |
| FIFO gross-финрез 2026 | Σ брокерского поля `yield` по сделкам 2026 | **PASS, Δ = 0.00** |

### Независимая верификация (оракул-агент, 2026-07-02)

- **Детерминизм**: повторный прогон `tax_alpha_scan.py` → все числа идентичны между запусками.
- **Per-сделка кросс-проверка** независимой реализацией знакового FIFO (не импорт
  функции сканера): **все закрытия 2026** сходятся с брокерским `yield`
  **по каждой сделке отдельно**, Δ=0.00 на каждой, на обоих счетах. Открытые лоты — бит-в-бит
  с `GetPositions` по обоим счетам.
- **Правовая логика** сверена с методичкой tax-loss-harvesting (RF-секция) и
  `deep/tax_optimization_iis3.md`: корзины ст. 214.1 (ЦБ↔фондовые ПФИ, НЕ
  валютные/товарные), wash-sale отсутствует, прогрессия 13–22% к суммарной базе,
  купоны в базе ЦБ, дивиденды отдельно — всё корректно.
- **Исправлены латентные баги** (на числа этого счёта не влияли, вскрыты кодревью, а не по факту):
  1. `ldv_eligible` зависел от `not has_iis` — само наличие ИИС-счёта ошибочно
     лишало бы ЛДВ ВСЕ лоты брокерского счёта (льготу теряют только бумаги,
     побывавшие на ИИС). После открытия ИИС-3 следующий скан дал бы неверный результат.
  2. `ldv_reached`/`ldv_ok` считались как `held >= 3*365 дней`, а `ldv_date` —
     календарные +3 года: через високосный год расхождение в 1 день
     (ложная «ЛДВ ДОСТИГНУТА» на 1095-й день). Теперь везде календарный рубеж.
  3. Крэш `ValueError` для лота, купленного 29 февраля (`date.replace(year+3)`) —
     добавлен `add_years` (29.02 → 01.03).
  4. Дубли строк в JSON `issues` (одна и та же оговорка попадала несколько раз) — дедуп.
  5. Harvesting не предупреждал об обнулении таймера ЛДВ при фиксации убытка
     по лоту, держащемуся ≥2.5 лет (шаг 3 TLH-алгоритма из deep-ресёрча) —
     добавлен флаг `ldv_timer_reset_warning` + предупреждение в рекомендации.

## Счета

Два счёта проверены: брокерский (`ACCOUNT_TYPE_TINKOFF`) и Инвесткопилка
(`ACCOUNT_TYPE_INVEST_BOX`), оба открыты в 2025 году.

ИИС-счетов нет → купленные здесь бумаги на ИИС не бывали, право на ЛДВ не потеряно.
Лотов из переводов бумаг (`INPUT_SECURITIES`) нет — происхождение всех лотов известно.

## [4] Реализованная база 2026 (FIFO gross = yield брокера)

Все закрытия года — продажи фондов денежного рынка, все в плюс. FIFO-реконструкция
gross-результата, комиссий и чистой базы сошлась с брокерским учётом до копейки на каждой
сделке (см. раздел «Сверка с оракулами» выше). Отдельно удержан НДФЛ по январскому
перерасчёту за предыдущий год — к базе текущего года не относится.

Совокупный годовой инвестдоход — далеко от порога прогрессии (2.4 млн ₽) → применяется
базовая ставка 13%.

Эффективная комиссия из истории использована в расчёте издержек «продажа+откуп» (см.
`commission_validate_result.md` — методология там же).

## [3] Открытые позиции (FIFO-лоты)

Единственная бумага в портфеле на момент прогона — вечный фонд денежного рынка,
несколько лотов, купленных в течение 2026 года, все лоты в небольшом плюсе к текущей цене.
Облигаций нет → НКД-нюанс в этом прогоне не задействован (в скрипте учтён: цена облигации
clean, НКД при продаже добавляется в базу как купонный доход).

## [5а] Tax-loss harvesting

**Кандидатов НЕТ**: ни одной открытой позиции с нереализованным убытком на момент прогона.
Сальдировать прибыль года нечем — экономия 0 ₽.

Механика (проверена и зашита в скрипт на будущее): wash-sale в РФ НЕТ (ст. 214.1 НК),
продажа+мгновенный откуп фиксирует убыток без потери экспозиции; порог целесообразности
`экономия (убыток × 13%) > издержки (2 × комиссия + полспреда из стакана)`;
расчёты по T+1 — фиксация не позднее ~28–29 декабря.

## [5б] ЛДВ-рубежи

Счёту меньше 2 лет — до первого 3-летнего рубежа далеко для всех лотов. При характере
позиции (фонд денежного рынка, используется как кошелёк с регулярными продажами) ЛДВ-
дисциплина практического смысла почти не имеет: FIFO при каждой продаже режет самые
старые лоты, дожить до 3 лет им, как правило, не суждено. Оценка «экономии на кону»
по текущим ценам — на уровне нескольких десятков рублей, то есть пренебрежима.

**Оговорка**: этот фонд торгуется на площадке `spb_etf_ru` — для ЛДВ требуется обращение
на организованном рынке РФ; применимость ЛДВ к этому листингу проверить у брокера
(гочтча проекта: Инвесткопилка — внебиржевой TMON OTC).

## [5в] Корзины сальдирования (ст. 214.1 НК)

Все инструменты истории — корзина **ЦБ** (акции, БПИФ/ETF, фонды денежного рынка).
Деривативов нет вовсе, конфликтов корзин нет:

- ЦБ ↔ фондовые ПФИ (фьючерсы на акции/индексы) — сальдируются;
- валютные/товарные ПФИ — только между собой, с ЦБ НЕЛЬЗЯ;
- ИИС не сальдируется с брокерским (ИИС отсутствует);
- между брокерами — только через 3-НДФЛ.

## Итоговые рекомендации (ручные действия)

1. **Harvesting сейчас: делать нечего** — убыточных позиций нет. Вернуться
   к скану в декабре 2026: перед фиксацией прибыли/убытков перезапустить
   `python scripts/tax_alpha_scan.py`.
2. **ЛДВ**: держать текущие лоты до истечения 3 лет спасает пренебрежимо малую сумму
   при таком размере позиции; плюс применимость ЛДВ к листингу `spb_etf_ru` под вопросом.
   Не фактор для решений.
3. **Главная доступная налоговая альфа — вне рынка: ИИС-3 отсутствует.** Пополнение
   ИИС-3 до 400 000 ₽/год даёт вычет на взнос до **52 000 ₽/год** (13% при наличии
   облагаемого НДФЛ дохода) — на порядки больше прочей альфы такого небольшого портфеля.
   Ловушка: бумаги, побывавшие на ИИС, навсегда теряют ЛДВ — кандидатов на ЛДВ
   на ИИС не переводить; фонды денежного рынка переводить безопасно.
4. Масштаб: при текущем размере портфеля любая оптимизация внутри него — копейки;
   следить за порогом прогрессии 2.4 млн ₽ на этом счёте пока не нужно.

## Оговорки

- Удержанный НДФЛ в текущем году может включать перерасчёт за предыдущий год
  (январское удержание брокером) — не путать с базой текущего года.
- Экономия ЛДВ посчитана по текущему нереализованному P&L — со временем изменится.
- Комиссия откупа взята как средняя историческая; полспреда — из стакана
  глубины 1 на момент прогона.
- Скрипт универсален (облигации/фьючерсы/переводы бумаг обрабатываются), но на этом
  счёте таких инструментов нет — ветки не прогреты реальными данными.
