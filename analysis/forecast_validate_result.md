# Согласованность консенсус-прогноза аналитиков

**Дата:** 2026-06-19 · `analysis/forecast_validate.py` · READ-ONLY · 8 голубых фишек

## Оракул
Поля `consensus.*` из `GetForecastBy` против списка индивидуальных таргетов инвестдомов.

## Результат: 46/48 арифметических тождеств
По всем бумагам **точно** выполняются:
- `minTarget` = min таргетов, `maxTarget` = max таргетов;
- `priceChange` = consensus − currentPrice; `priceChangeRel` = 100·priceChange/currentPrice;
- `consensus` = среднее таргетов — **7/8** (SBER 396.5, GAZP 160.1, LKOH 6438, ROSN 558.9,
  TATN 734.7, MTSS 267.1, PLZL 2921);
- арифметика по каждому таргету (priceChange = target − currentPrice).

## Исключение: GMKN
`consensus` 178.0 vs среднее таргетов 179.08, и арифметика по таргетам 0/13. Причина —
`currentPrice` в consensus (137.92) НЕ согласован с currentPrice, относительно которого посчитаны
строки таргетов GMKN (рассинхрон снимка цены конкретно у этой бумаги). Остальные поля (min/max/Δ)
у GMKN корректны.

## Главная находка: рекомендация ≠ голоса и ≠ апсайд
Рекомендация консенсуса **НЕ выводится** ни из простого большинства голосов, ни из порога апсайда:
- **GAZP**: апсайд **+49.5%**, голоса BUY 7 / HOLD 5 / SELL 1 (плюрализм BUY) — но `rec = HOLD`;
- **MTSS**: апсайд +18.7% → HOLD; **SBER**: апсайд +25.7% → BUY (порог не монотонен);
- т.е. GAZP с бóльшим апсайдом, чем SBER, имеет более осторожную рекомендацию.

Вывод: `recommendation` — отдельный (вероятно проприетарный) сигнал, его нельзя восстановить ни
из таргетов, ни из апсайда. Числовые агрегаты (среднее/min/max/изменение) — детерминированы и
воспроизводимы; рекомендацию принимать как есть, не пытаясь вывести из чисел прогноза.

## Практический вывод
Консенсус-таргет = среднее прогнозов инвестдомов (можно пересчитать/взвесить самому), диапазон =
min/max, апсайд = к текущей цене. А вот текстовую рекомендацию BUY/HOLD/SELL нельзя выводить из
этих чисел — это независимый сигнал.
