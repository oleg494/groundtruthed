# Инварианты стакана и кросс-эндпоинт «последняя цена»

**Дата:** 2026-06-19 (премаркет) · `analysis/orderbook_validate.py` · READ-ONLY · 6 ликвидных акций

## Оракулы
Структурные (обязаны выполняться всегда) + кросс-эндпоинт согласованность.

## Результат
**Структурные инварианты — 24/24 идеально** по всем 6 бумагам:
- bids убывают по цене, asks растут (упорядоченность);
- книга не пересечена: best_bid < best_ask;
- ценовой коридор: limitDown ≤ best_bid, best_ask ≤ limitUp.

Это реальная проверка целостности рыночных данных: ни одной перевёрнутой/пересечённой книги,
все котировки внутри биржевых лимитов планки.

**Кросс-эндпоинт «последняя цена» — с точностью до снимка:**
- `OrderBook.lastPrice == GetMarketValues LAST_PRICE`: 5/6;
- `последняя сделка (GetLastTrades) ≈ lastPrice`: 4/6.

Три расхождения — НЕ противоречие данных, а тайминг: каждый из трёх методов (стакан, market
values, лента) — отдельный вызов в живом премаркете, снимки сделаны в разные миллисекунды,
цена между ними успевает измениться. На спокойном инструменте (GAZP, VTBR, GMKN) все три
источника совпали.

## Вывод
Стаканы корректны структурно всегда (упорядочены, не пересечены, в коридоре планок). «Последняя
цена» из разных методов API — это одно и то же число с точностью до момента снимка; для строгой
сверки брать значения одним вызовом/в один момент, а не последовательными запросами по живому
рынку.
