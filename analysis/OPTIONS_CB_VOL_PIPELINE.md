# Опционный конвейер вокруг заседаний ЦБ

Дата: 2026-07-05
Статус: план исследования, без заявок, без портирования в ферму.

## Почему это имеет смысл

Дельта-один календарная стратегия `CALENDAR_CB` убита: направленного дрейфа нет, DSR низкий,
профит концентрируется в единичных сюрпризах. Но отдельное измерение
`analysis/botdev2_calendar_cb_measure.py` подтвердило другой факт: реализованная волатильность
сжимается перед заседанием и расширяется в день 0/+1. Это нельзя монетизировать акциями/фьючерсами
на направление, но можно проверять опционными конструкциями.

## Существующий оракул

`analysis/options_parity_validate.py` уже вскрыл конвенцию THEOR_PRICE:

- опционы premium-style, европейские, cash-settled;
- паритет: `C-P = D*(F-K)`;
- `F_impl = K+C-P = D*F + (1-D)*K`;
- наклон по страйку даёт дисконт-фактор, implied-ставку и форвард;
- THEOR_PRICE пригоден даже при отсутствии сделок, но это модельная цена биржи, не гарант исполнения.

## Минимальный исследовательский pipeline

1. **Chain snapshot**
   - Вход: базовый актив с опционной цепочкой и ближайшая экспирация, покрывающая заседание ЦБ.
   - Данные: `OptionsBy` или fallback `Options`+локальная фильтрация, `FutureBy`,
     `GetMarketValues(THEOR_PRICE, LAST_PRICE)`.
   - Оракул: raw parity-fit сохраняется как диагностика; решение качества — по MAD-filtered
     robust fit `R² >= 0.999`, `pairs >= 20`, outliers не больше `max(1, 5% пар)`.
   - Если robust fit хороший, но raw fit плохой, snapshot пригоден для поверхности только после
     явного исключения outlier-страйков из последующих IV/vol метрик.
   - Дополнительные проверки: бабочка неотрицательна, IV call/put на одном страйке сходится.

2. **Event window**
   - Для каждого планового заседания взять снимки `k=-5,-3,-1,0,+1,+3`.
   - Не использовать внеочередные заседания: они не антиципируются календарно.
   - Если исторических option snapshots нет, начать с forward-накопления, а не симулировать прошлое из текущего смайла.

3. **Vol metric**
   - ATM IV: интерполяция по ближайшим страйкам около восстановленного форварда.
   - Skew: `IV_put_25d - IV_call_25d` или упрощённо разница крыльев на равном moneyness.
   - Term effect: IV ближайшей экспирации минус следующей, если обе покрывают событие.

4. **Tradable constructs**
   - Long straddle/strangle перед заседанием: прибыль требует, чтобы realized move + post-event IV не съели theta/spread.
   - Calendar spread: если ближняя event-IV переоценивается относительно дальней.
   - Directional options запрещены как основной тест: это снова CALENDAR_CB под другим названием.

5. **Cost model**
   - Использовать bid/ask, если доступен стакан; THEOR_PRICE только для оракула поверхности.
   - Отдельно учитывать комиссию, шаг цены, гарантийное обеспечение, проскальзывание и невозможность market orders по опционам.
   - Reject rule: если expected edge меньше 2*spread + комиссия + запас на theta, конструкция не тестируется.

6. **Decision criteria**
   - Не меньше 30 независимых event-trades для статистического вывода. Если событий меньше, результат только measurement, не strategy.
   - Бить cash benchmark и buyhold-vol proxy после издержек.
   - DSR/PSR считать по сделкам, а не по дневным барам, и явно штрафовать число перебранных конструкций.

## Первый кодовый шаг

Создать `analysis/options_cb_snapshot.py`:

- read-only snapshot одной цепочки;
- reuse-подход из `options_parity_validate.py`;
- выход `analysis/options_cb_snapshot.json` и краткий markdown;
- fallback с `OptionsBy` на broad `Options`, потому что live REST `OptionsBy` для Si-9.26
  вернул HTTP 400 `30102` при валидном `basicAssetPositionUid`;
- robust parity diagnostics (`fit`, `robust_fit`, `outliers`);
- без бэктеста и без торговых сигналов.

Только после накопления нескольких живых snapshots вокруг заседаний переходить к
`analysis/options_cb_event_study.py`.

## Kill conditions

- Нет ликвидного стакана на ATM/near-ATM страйках.
- THEOR_PRICE безарбитражен, но bid/ask делает straddle edge отрицательным.
- Эффект IV виден только ex post в realized volatility, но не в pre-event implied volatility.
- Положительный результат держится на одном заседании-сюрпризе.
