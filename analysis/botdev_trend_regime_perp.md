# TREND_REGIME_PERP — трендовая на вечных фьючерсах с режимным фильтром

Дата: 2026-07-02. Кандидат конвейера botdev. Статус: **завершён, вердикт KILL**.

## Гипотеза

Donchian-тренд на вечных фьючерсах GLDRUBF и IMOEXF (ролловер не нужен, expdate=2099)
прибылен, если торговать ТОЛЬКО в трендовом режиме: Hurst>0.55 И ADX>порог
(пороги из `deep/market_regime_moex.md`: H(100)>0.55 + ADX>25 = режим тренда).

## Реализация

- `backtest/strategies.py` → `RegimeDonchian` (`regime_donchian`): был ADX-гейт,
  **добавлен опциональный Hurst-гейт** (`hurst_n`, `hurst_min`; `hurst_min=0` = выключен,
  прежнее поведение не изменено). Индикатор `ta.hurst` (R/S) уже был в `indicators.py`.
- `backtest/__main__.py`: команды `optimize`/`walkforward`/`study` теперь прокидывают
  `--futures --multiplier` в движок (раньше только `run`) — иначе фьючерсная маржинальная
  модель в study не работала.

Sanity-check `ta.hurst` на известных процессах (реальный stdout):

```
RW: 0.4526 (n=100)  0.4946 (n=256)      # случайное блуждание → ~0.5 ✓
persist(rho=0.6): 0.683 / 0.800         # персистентный → >0.5 ✓
antipersist(rho=-0.6): 0.534 / 0.500    # слабо различает вниз (R/S на коротких окнах смещён вверх)
```

Вывод: гейт H>0.55 на n=100 дневных барах — жёсткий (медленный тренд типа `trend:800:3`
даёт H≈0.24 из-за низкого отношения дрейф/шум на лог-приростах); сетка включает
hurst_min=0 (выключен), 0.5, 0.55 — пусть данные решают.

## Замечания по ходу

- Дневной таймфрейм даёт 2–5 сделок на весь период → OOS-окна пустые/шумные,
  статистическая мощность около нуля. Часовые бары (6.7 тыс.) — основная надежда.
- IS→OOS ratio 0.34 (GLD) и -0.49 (IMOEXF) — классическая переоптимизация.
- Hurst-гейт в WFA выбирался лишь в 1 окне из 8 (GLD окно 2, и там OOS провалился)
  — на этих данных фильтр ценности пока не показал.

## Вердикт: KILL

Критерии (все обязательны):

| критерий | GLD день | GLD час | IMOEXF день | IMOEXF час |
|---|---|---|---|---|
| OOS > 0 | +18.39% ✓ | **-4.46% ✗** | **-9.72% ✗** | **-2.13% ✗** |
| DSR > 0.5 | 76.1% ✓ | **12.0% ✗** | 74.4% ✓* | **26.0% ✗** |
| бьёт buyhold | **✗** (+48 vs +76) | **✗** (+4.2 vs +14.5) | ✓ | ✓ |
| бьёт random | ✓ (+48 vs +36) | **✗** (+4.2 vs +8.2) | ✓ | ✓ |

\* дневные DSR посчитаны по IS-кривой — при отрицательном/непобеждающем OOS не спасают.

Ни одна из четырёх конфигураций не проходит все критерии одновременно. Единственный
плюс (GLD день, OOS +18.4%) держится на 5 сделках и проигрывает buy&hold золота почти
вдвое; на часовиках, где сделок достаточно (27–63), эдж исчезает полностью (6–7%
положительных ячеек сетки, DSR 12–26%). Hurst-гейт статистической ценности не показал:
на дневках WFA его почти не выбирал, на IMOEXF-часах выбирал — и OOS всё равно отрицателен.

Гипотеза «Donchian-тренд на вечных фьючах в трендовом режиме (Hurst+ADX)» — **отклонена**.
Урок для пула: режимный фильтр сокращает число сделок до статистической ничтожности на
дневках и не добавляет эджа на часах; лонг-онли тренд на золоте в 2023–2026 просто хуже
владения золотом (тренд был почти непрерывный — любой выход вредил).
