# Откуда брать и как обрабатывать информацию о рынке

Дата: 2026-06-11. Контекст: новость «МосБиржа падает 14 недель подряд» — где брать
такое самому и что с этим делать. NotebookLM пробовали — слабоват для живого рынка
(статичная база, ручная загрузка, 50 запросов/день).

## Главная мысль: 80% «новостей» — это вычислимые факты

Новость про 14 недель мы **проверили за 20 строк кода** по недельным свечам IMOEX:
ровно 14 красных недель, −16,5% от максимума. Никакой журналист не нужен —
нужен скрипт. Рыночная информация делится на два сорта, и обрабатывать их надо по-разному:

| Сорт | Примеры | Источник | Обработка |
|---|---|---|---|
| **Факты** (вычислимые) | серии падения, дистанция от хаёв, сдвиг кривой ОФЗ, RUSFAR vs ставка, дивгэпы, объёмы | T-Invest API, MOEX ISS | скрипты → таблицы/алерты, ноль чтения |
| **Нарративы** (события) | решения ЦБ, санкции, отчёты эмитентов, дивидендные рекомендации, геополитика | пресс-релизы ЦБ, e-disclosure, РБК/Интерфакс RSS | LLM-суммаризация (Claude в этой сессии) |

## Источники по убыванию ценности

### 1. T-Invest API (уже есть, бесплатно)
Свечи всех таймфреймов, `GetAssetReports` (календарь отчётностей!), `GetDividends`
(объявленные дивиденды), `GetConsensusForecasts` (что думают аналитики),
`GetInsiderDeals`. Этого хватает на 80% «фактового» слоя.

### 2. MOEX ISS API (бесплатно, без токена)
`iss.moex.com` — официальные данные биржи: составы индексов, обороты по режимам,
короткие позиции, данные срочного рынка (открытый интерес юр/физлиц — единственный
легальный «сентимент» розницы vs профи). JSON, без регистрации.

### 3. ЦБ РФ (бесплатно, критично для стратегии пользователя)
- `cbr.ru/scripts/XML_daily.asp` — курсы;
- ключевая ставка и **календарь заседаний** — со страницы ЦБ;
- пресс-релизы по ставке — RSS. Для держателя фонда денежного рынка это
  единственный источник, который реально двигает его доходность.

### 4. RSS-ленты деловых СМИ (бесплатно)
Интерфакс, РБК, Коммерсантъ отдают RSS. Заголовков достаточно: LLM фильтрует
по релевантности (ставка/ОФЗ/эмитенты из портфеля), полные тексты не нужны.

### 5. NotebookLM — переквалифицировать
Не для живого рынка, а для **корпуса знаний**: чат-история API (уже там),
методички, годовые отчёты эмитентов. То, что не устаревает за день.

## Предлагаемая конструкция (когда дойдут руки)

`scripts/market_context.py` — «фактовый» слой одной командой:
- серии недель/дней по IMOEX/RGBI, дистанция от 52w-хаёв;
- сдвиг кривой ОФЗ за неделю (по ofz_screen.json историческим срезам);
- RUSFAR vs ключевая (рынок закладывает изменение ставки?);
- дни до заседания ЦБ (календарь хардкодом: 25.07, 12.09, 24.10, 19.12 — 2026);
- выход: markdown в analysis/ + цветной терминал.

`scripts/news_pull.py` — «нарративный» слой:
- стянуть RSS (urllib, без зависимостей), отфильтровать по ключам
  (ставка, ОФЗ, тикеры портфеля), сложить в analysis/news_YYYY-MM-DD.md;
- интерпретация — не кодом: открываешь Claude Code и просишь разобрать файл
  (методология morning-note из финансовых скиллов Anthropic — не входит в этот репозиторий).

Итого пайплайн: **API/ISS → скрипты → факты; RSS → файл → Claude → смысл.**
NotebookLM остаётся библиотекой, а не лентой.

## Что НЕ делать
- Не парсить телеграм-каналы (API-боль, шум, выживший контент).
- Не платить за терминалы/данные — на этом размере капитала не окупается.
- Не строить «реалтайм»: для стратегии «фонд + ОФЗ» достаточно дневного среза.
