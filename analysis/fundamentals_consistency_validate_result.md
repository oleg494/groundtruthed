# Внутренняя согласованность фундаментальных мультипликаторов

**Дата:** 2026-06-19 · `analysis/fundamentals_consistency_validate.py` · READ-ONLY · 8 голубых фишек

## Задача и оракул
`GetAssetFundamentals` отдаёт и сырые цифры (капитализация, акции в обращении, прибыль, выручка,
FCF, дивиденд на акцию, EPS), и готовые коэффициенты (P/E, P/S, P/FCF, маржа, дивидендные).
Оракул — поля-коэффициенты. Проверяем, что они согласованы с реконструкцией из сырых цифр того
же ответа, плюс кросс-эндпоинт: подразумеваемая цена `marketCap/shares` = рыночная (last price).

## Результат: 52/52 тождеств сошлись
SBER, GAZP, LKOH, ROSN, GMKN, TATN, PLZL, MGNT — все проверяемые мультипликаторы совпали
(допуск под округление полей до 2 знаков + 1% относительный).

Проверенные тождества (по каждой бумаге — те, где поля ненулевые):
| тождество | смысл |
|---|---|
| `price = marketCap / sharesOutstanding` | капитализация ÷ число акций = рыночная цена |
| `EPS = netIncomeTtm / sharesOutstanding` | прибыль на акцию |
| `P/E = marketCap / netIncomeTtm = price / EPS` | цена/прибыль |
| `P/S = marketCap / revenueTtm` | цена/выручка |
| `P/FCF = marketCap / freeCashFlowTtm` | цена/свободный денежный поток |
| `netMargin = netIncomeTtm / revenueTtm` | чистая маржа |
| `divPayout = dividendsPerShare / EPS` | коэффициент выплат |
| `divYield = dividendsPerShare / price` | дивидендная доходность |

Поля, равные 0 (не раскрыты эмитентом — напр. EBITDA, ROIC у части компаний; P/E у LKOH/MGNT),
из проверки исключаются: проверяем только то, что API реально отдал.

## Замечания
- Коэффициенты на базе MRQ (например `roe`, `priceToBook`) НЕ образуют точного тождества с
  TTM-прибылью (разные периоды: балансовые показатели на конец квартала против скользящей
  годовой прибыли) — поэтому в проверку не включены, чтобы не сравнивать разные базы.
- `price = marketCap/shares` совпала с текущей рыночной ценой по всем 8 бумагам — значит
  капитализация в фундаментале пересчитывается по актуальной котировке (а не по устаревшей).

## Практический вывод
Мультипликаторы из `GetAssetFundamentals` можно использовать как есть — они арифметически
согласованы с сырыми показателями. При этом, зная сырьё, любой коэффициент пересчитывается
локально под свой сценарий цены/прибыли (напр. форвардный P/E под прогноз прибыли) с гарантией
той же методики, что у API.
