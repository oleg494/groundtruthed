# GetMaxLots: согласованность лимитов заявок

**Дата:** 2026-06-19 · `analysis/maxlots_validate.py` · READ-ONLY (GetMaxLots — read-метод) · брокерский счёт

> Числа этого прогона (позиции, доступные средства) относятся к реальному счёту автора и
> не публикуются — здесь только методология и проверенные соотношения.

## Оракулы и результат — 9/9 PASS
| проверка | результат |
|---|---|
| sellMaxLots (без маржи) == позиция/лот | ✓ (для бумаг без позиции — 0) |
| buyMoneyAmount / (цена·лот) ≈ buyMaxLots | ✓ (с точностью до округления/плеча, ~0.7%) |
| buyMaxMarketLots ≤ buyMaxLots | ✓ |

## Вскрытые соотношения
- **Продажа без плеча = ровно текущая позиция**: для бумаги в портфеле лимит равен числу
  лотов позиции, для бумаг без позиции — 0. Маржинальная продажа (`sellMarginLimits`)
  больше — включает шорт под плечо.
- **Денежный лимит покупки соответствует числу лотов по текущей цене** (с учётом размера
  лота инструмента): `buyMoneyAmount/(price·lot) ≈ buyMaxLots` с точностью до плеча/округления.
- **Рыночных лотов чуть меньше лимитных** — резерв на проскальзывание при market-заявке
  (наблюдалось на уровне единиц процентов).

## Вывод
GetMaxLots внутренне непротиворечив и пригоден для расчёта покупательной способности: продать
можно ровно позицию (или больше под шорт-плечо), купить — на доступные деньги (с плечом) по
текущей цене. Цифры согласованы с позициями (`GetPositions`), ценой (`GetMarketValues`) и лотностью
инструмента. (Примечание: на счёте Инвесткопилки GetMaxLots возвращает INTERNAL для авто-инструмента
— зафиксировано как ограничение.)
