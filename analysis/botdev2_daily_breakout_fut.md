# BOTDEV2 — DAILY_BREAKOUT_FUT (пробой дневного Дончиана на 30-мин фьючерсах)

**Вердикт: KILL.** Дата: 2026-07-02. Конвейер: `analysis/botdev2_daily_breakout_fut.py`
(smoke → grid 27 → anchored WFA ×4 → DSR → бенчмарки). Прогон детерминирован:
перегон run2 совпал с run1 бит-в-бит (`diff` пуст → DETERMINISM_OK).

## Гипотеза (инверсия kill-пула первой волны)

Первая волна убила оба УТРЕННИХ режима (ORB и контртренд ORB, `analysis/botdev_orb_reversal.md`):
частота ложных утренних пробоев не монетизируется ни по тренду, ни против. Кандидат —
другой класс: пробой **N-дневного** канала Дончиана (10–20 дней), исполняемый внутри дня
на 30-мин барах, позиция живёт дни (ночёвки есть, в отличие от daybot). Вход — close
30-мин бара за пределами канала N завершённых дней (исполнение по open следующего бара,
без lookahead); выход — обратный канал M<N и/или трейлинг trail_mult×ATR(14) по дневным
агрегатам. Обе стороны (фьючерсы).

Реализация: класс `DailyBreakoutFut` (`daily_breakout`) в `backtest/strategies.py`,
зарегистрирован в REGISTRY (24 стратегии, существующие не тронуты).

## Данные и модель издержек

30-мин свечи из кэша `backtest/.cache` (sandbox-домен): BMQ6 3451 / NGN6 3176 /
GLDRUBF 4625 баров, 2026-01-05 → 2026-07-01 (~120 торговых дней — молодые контракты,
та же оговорка о статистической силе, что у первой волны). Комиссия 0.05% + слиппедж
0.05% (как у первой волны, для сопоставимости). `kind='futures'`,
multiplier = point_rub из `lab/instruments.py` (BMQ6 71.908, NGN6 7190.77, GLDRUBF 1.0).
CASH 100 000. Сетка: n ∈ {5,10,15,20} × exit_m ∈ {3,5,10} × trail_mult ∈ {0,2,3},
валидных комбинаций (exit_m<n) — **27 испытаний на инструмент**.

## Smoke на синтетике (благоприятный мир: многодневные тренды по ~40 дней)

```
SMOKE synth_multiday_trend(160 дн): сделок 5 (лонгов 2, шортов 3, многодневных 5), ret +32.34%, WR 80%, PF 15.31
SMOKE с трейлом 2.0×ATR: сделок 5, ret +34.57%
```

Стратегия торгует в обе стороны, держит позиции через ночь, зарабатывает там, где
должна — реализация корректна, дальше вопрос только к эджу на реальных данных.

## Тройной критерий выживания

| Критерий | BMQ6 | NGN6 | GLDRUBF |
|---|---|---|---|
| Сквозной OOS > 0 | **+4.38%** ✓ | −11.22% ✗ | −11.65% ✗ |
| DSR > 0.5 | 79.6% ✓ (но по IS-кривой) | 16.4% ✗ | 27.7% ✗ |
| Бьёт buyhold И random | ✗ (bh **+16.77%** > +4.38%; random +1.12% побит) | ✗ (проигрывает и bh −10.39%, и random −6.25%) | ✗ (проигрывает bh −8.46%, random −8.43%) |

Ни один инструмент не проходит все три критерия. **KILL.**

## Диагноз (не только симптом)

1. **Единственный «плюс» (BMQ6 OOS +4.38%) — не эдж, а разбавленный buyhold**: за 4
   OOS-окна закрыто всего 2 сделки; профит дали два переноса позиции по июньскому
   тренду Brent, и даже так стратегия отдала владению контрактом 12.4 п.п.
   (+4.38% vs +16.77%). Любой выход из тренда по каналу/трейлу только вредил — тот же
   диагноз, что у TREND_REGIME_PERP на золоте в первой волне.
2. **NGN6: эджа нет даже in-sample** — все 27 точек сетки убыточны (доля положительных
   0%), лучшая −12.37%. Это не переоптимизация, это отсутствие гипотезы на инструменте:
   газ 2026 пилит дневной канал ложными пробоями в обе стороны (22 сделки, WR 27%).
3. **GLDRUBF: золото 2026 — боковик внутри канала**; редкие пробои (10 сделок IS)
   дают PF 1.17 — съедается первым же неудачным OOS-окном. Три из четырёх OOS-окон
   в минусе.
4. Класс «позиционный пробой vs интрадей» инверсию не спас: на ~120 днях молодых
   фьючерсов дневной Donchian даёт 0–4 сделки на OOS-окно (≪30 по правилу
   `.agents/AGENTS.md`) — вердикт держится на единодушии: 2 из 3 инструментов в минусе
   по OOS, 3 из 3 не бьют buyhold, DSR ниже порога везде, где есть сделки.

## Оговорки

- История коротка (~6 мес), сделок мало — но это симметрично работает и против
  «выжившего» BMQ6: его плюс статистически пуст. Ждать больше истории ради стратегии,
  которая на лучшем инструменте проигрывает buyhold, смысла нет.
- DSR BMQ6 79.6% посчитан по IS-кривой лучшей точки сетки (как в первой волне у
  TREND_REGIME_PERP) — при непобеждённом buyhold на OOS не спасает.
