# Отчёт сессии 2026-06-19: проверки-оракулы T-Invest API

Автономная сессия (~2 часа). Задача от пользователя: придумывать и решать задачи, каждая — с
**объективным проверяемым оракулом**. Тема: что может LLM в харнесе с беспощадной проверкой.

## Что сделано
**28 проверок-оракулов** (+ регресс-раннер), все закоммичены. Каждая сверяет данные/вычисления
T-Invest против объективного эталона — где возможно, бит-в-бит. Полная карта: `VALIDATION_INDEX.md`.
Запуск набора: `python analysis/run_all_validations.py` → 26 PASS / 2 WARN / 0 FAIL.

## Главное: вскрытые недокументированные конвенции
Не «реализуй формулу», а **реверс-инжиниринг конвенций против числового оракула**:
- **Тех-анализ** (2463/2464 бит-в-бит): RSI=сглаживание Уайлдера, BB=популяционная дисперсия
  (÷N), EMA seed-инвариантна при прогреве, AVG=(o+h+l+c)/4.
- **Опционы FORTS**: premium-style паритет С ДИСКОНТОМ (R²=0.999996) → рублёвая ставка 13.56% и
  форвард из наклона по страйку. Первая гипотеза (futures-style) была отвергнута оракулом.
- **Облигации**: YTM = эффективная годовая, ACT/365, расчёты T+1 (±1.3 б.п. к полю yield).
- **Свечи**: граница дня = MSK-календарь; старший ТФ = бит-в-бит свёртка младшего.
- **Бета**: дневные доходности за 1 год vs IMOEX.
- **Комиссия**: тариф 0.0593% (брокерский) / 0% (копилка).
- **CIP**: фьючерсы и опционы взаимно согласованы (r_usd≈3%).

## Вскрытые грабли (в docs/gotchas.md)
- НКД из `GetAccruedInterests` не наивно-линеен (T+1 + заморозка на выходных).
- ГО фьючерса ≠ нотионал × dlongClient (брать GetFuturesMargin).
- Сырой REST НЕ кладёт строку `value` в денежные поля — парсить units/nano.
- Инвесткопилка торгует через внебиржевой `TMON OTC` (нет свечей, GetMaxLots=INTERNAL).
- `volumeBuy+volumeSell ≠ volume` у акций в общерыночные аукционные дни (95% на общих датах).
- close из `GetLastTrades` не восстановить (секундное разрешение меток).
- Рекомендация консенсус-прогноза — не из голосов и не из апсайда (отдельный сигнал).

## Метод
Дисциплина «оракул решает всё»: где гипотеза не угадана (futures-style паритет, sample-дисперсия
BB, наивный НКД) — оракул это показал, и конвенция исправлялась/документировалась, а НЕ
подгонялась под «успех». Пороги не задирались ради красивых чисел (напр. в тех-анализе единственный
промах честно классифицирован как ничья округления, а не спрятан). Отрицательные и частичные
результаты (WARN) зафиксированы наравне с успехами.
