# Инварианты агрегации свечей — старший ТФ = свёртка младшего

**Дата:** 2026-06-19 · `analysis/candle_aggregate_validate.py` · READ-ONLY (GetCandles) · SBER

## Задача и оракул
Старший таймфрейм обязан быть точной OHLCV-свёрткой младшего: open=первая, close=последняя,
high=max, low=min, volume=сумма. Оракул — сами серверные свечи старшего ТФ. Проверка
детерминированная и бит-в-бит. Попутно вскрываем недокументированные конвенции бакетирования.

## Результаты
| свёртка | результат |
|---|---|
| **недельные ← дневные** | **55/55 бит-в-бит** |
| **месячные ← дневные**  | **11/11 бит-в-бит** |
| **дневные ← часовые (регулярные сессии)** | **12/12 бит-в-бит** |
| дневные ← часовые (выходные/праздники) | НЕ собираются — см. ниже |

## Вскрытые конвенции
1. **Граница дня = MSK-календарь (UTC+3), не UTC.** Группировка по UTC-дню даёт больше
   расхождений (8 против 6 на том же окне). API маркирует свечи в UTC, но режет сутки по
   московской полуночи: дневная свеча с меткой `…T00:00:00Z` = торговый день этой даты по MSK.
2. **Дневная свеча будней = весь MSK-день**, включая премаркет (≈06:00–09:00 MSK), основную
   сессию и вечёрку (19:00–23:50). Все 12 регулярных будних сессий собрались из часовых
   точно по OHLCV.
3. **Неделя — ISO-неделя (пн–вс) по MSK; месяц — календарный по MSK.** Обе свёртки из дневных
   бит-в-бит на полных бакетах.

## Вскрытая грабля: выходные/праздничные дневные свечи
Дневные свечи за **субботу/воскресенье и праздники** (в окне: 31 мая, 6–7, 13–14 июня; 12 июня —
День России) **нельзя собрать из часового потока** — расходятся и по объёму, и местами по OHLC.
Пример: суббота 06-06 — сумма часовых ΣV=1 460 053, а дневная V=1 373 638 (Δ=86 415). Даже
ограничение часовых только основной сессией 10:00–18:59 не сходится по объёму (Δ≈900).

**Вывод:** выходные/праздничные дневные свечи MOEX формирует из ОТДЕЛЬНОГО пайплайна (иная
нарезка сессии «выходных торгов»), а не агрегацией того же часового ряда, что отдаётся в HOUR.
12 июня (пятница-праздник) ведёт себя как выходной — значит предиктор не «будний день», а
«регулярная торговая сессия».

## Практический вывод
- Любой старший ТФ можно строить локально из младшего и доверять ему **на регулярных сессиях** —
  экономит лимит marketdata (600/мин) и даёт произвольные кастомные интервалы.
- НО для выходных/праздников дневную свечу надо брать у API напрямую: локальная свёртка часовых
  даст другое значение. Это важно для бэктестов, захватывающих weekend-сессии MOEX.
- Все вычисления времени вести по MSK-календарю, а не по UTC-дате метки.
