# Знак payment vs тип операции

**Дата:** 2026-06-19 · `analysis/operation_sign_validate.py` · READ-ONLY · оба счёта, 400 операций

## Оракул и результат
Знак денежного потока операции обязан соответствовать её экономическому смыслу: списания
(покупка, комиссия, налог, вывод) — payment ≤ 0; зачисления (продажа, купон, дивиденд, ввод,
погашение) — payment ≥ 0.

**400/400 операций с корректным знаком** (по 200 на каждый счёт, 0 нарушений).

## Вывод
Денежный поток операций строго согласован с их типом — нет «перевёрнутых» знаков. Это базовый
инвариант целостности журнала операций, важный для любых сверок (account_reconcile, tax_reconcile,
commission_validate): суммировать payment можно напрямую, знак уже несёт направление потока.
Подтверждает корректность знаковой логики в реконструкциях счёта.
