# VOL_REGIME_SWITCH — vol-таргетирование buyhold-корзины (вторая волна bot-dev)

**Вердикт: KILL** (2026-07-02). Sharpe хуже buyhold во всех 4 OOS-окнах, сквозной
OOS −14.20%, random по Sharpe не побит. Единственное сдержанное обещание —
просадка мельче buyhold во всех 4 окнах — недостаточно по критериям выживания.

## Гипотеза

Инверсия диагноза ABSMOM_SWITCH (первая волна, kill): там переключение актив↔кэш
шло по ЗНАКУ доходности, и «эдж» оказался кэш-прокси (+86…+107 п.п. от ставки
14.25%). Здесь переключение — по РЕЖИМУ ВОЛАТИЛЬНОСТИ без прогноза направления:
экспозиция равновзвешенной корзины = min(1.0, target_vol / RV(N)), где RV —
годовая реализованная вола портфельного ретёрна (учитывает корреляции).
Классика Moreira & Muir (2017): vol-managed portfolio улучшает Sharpe владения.
**Кэш-прокси сознательно НЕ начисляется** — свободный кэш лежит мёртвым (0%),
чтобы не повторить артефакт absmom.

Честный бенчмарк — тот же равновзвешенный buyhold на той же корзине.
Критерий survive: сквозной OOS>0, DSR>0.5, Sharpe И maxDD лучше buyhold при
сопоставимом ret, OOS-устойчивость (ΔSharpe>0 в ≥половине окон), бьёт random.

## Реализация

- `backtest/strategies.py` → класс `VolRegimeSwitch` (`name="vol_regime_switch"`,
  в REGISTRY). Параметры: `lookback`, `target_vol`, `max_leverage=1.0` (только
  де-левередж), `rebalance`, `band=0.05` (мёртвая зона по экспозиции — душит
  комиссионный чёрн), `frac=0.99`. Ребаланс ждёт прогрева ВСЕЙ корзины.
- Конвейер: `analysis/botdev2_vol_regime_switch.py` (фетч → grid 32 → anchored
  WFA c warm-up → DSR → пооконный buyhold → вердикт).
- Абляция кэш-прокси: `analysis/botdev2_vrs_ablation.py`.

Свободный кэш ставки НЕ получает — по условию кандидата. Масштаб: у пооконных

Абляция (`botdev2_vrs_ablation.py`): тот же WFA, но кэшу капает 14.25% годовых

```

  окно 2: OOS ret   -4.46%  params {'target_vol': 0.1, 'lookback': 60, 'rebalance': 21}
  окно 3: OOS ret   +7.27%  params {'target_vol': 0.1, 'lookback': 60, 'rebalance': 21}
  окно 4: OOS ret  -10.36%  params {'target_vol': 0.1, 'lookback': 60, 'rebalance': 21}
```

Кэш-прокси переворачивает вердикт с −14.20% на +13.41% — **+27.61 п.п. из

воспроизводит диагноз ABSMOM_SWITCH на другой стратегии: при ключевой ставке
14.25% ЛЮБОЕ правило, паркующее капитал в кэш, выглядит «эджем», если кэшу
начислять ставку. Правильная планка для таких стратегий — не buyhold акций,
а фонд денежного рынка (LQDT ≈ ставка): +13.41% за ~2.7 года OOS ей проигрывает.

IS-точки глубже buyhold, random по Sharpe не побит, ΔSharpe>0 в 0/4 OOS-окон.

### Диагноз (почему vol-targeting не сработал на MOEX 2022–2026)

1. **Вола не предсказывает risk-adjusted доходность на этой ленте.** Премия
   vol-managed портфелей (Moreira–Muir) живёт там, где после всплесков волы
   идут слабые risk-adjusted ретёрны. На MOEX пост-2022 всплески волы часто
   completion-эпизоды падения с V-образным отскоком: де-левередж системно
   опаздывал — резал экспозицию на дне и недобирал отскок (окно 2: ΔSharpe
   −0.39 при Δdd +7.27 п.п. — просадку срезал, но и восстановление тоже).
2. **Без кэш-прокси оверлей на падающей корзине не может быть OOS-положительным:**
   стратегия не прогнозирует направление, а её единственный «актив» при низкой
   экспозиции — мёртвый кэш. Сквозной buyhold за тот же OOS-период ≈ −10.3%
   (компаунд окон), стратегия −14.20% — даже относительного выигрыша по
   доходности нет из-за комиссии на перекладках и запаздывания RV.
3. **DD-режущее свойство реально** (4/4 окон, до −9.3 п.п. просадки) — но это
   свойство риск-менеджмента, а не источник доходности. Для фермы бесполезно
   без эджа-носителя, к которому его можно прикрутить.

### Заметки на будущее (не обещания)

- Vol-оверлей имеет смысл тестировать не над buyhold-корзиной, а над стратегией
  с собственным эджем (когда такая появится) — как risk-слой, с метрикой
  «Sharpe оверлея vs Sharpe носителя».
- Планка для любых кэш-паркующих стратегий при КС 14.25% — денежный рынок
  (+14.3%/год), не buyhold акций. Критерий стоит закрепить в `.agents/AGENTS.md`.
