# Реверс-инжиниринг GetTechAnalysis — воспроизведение сервера бит-в-бит

**Дата:** 2026-06-19 · `analysis/techanalysis_validate.py` · READ-ONLY (GetCandles + GetTechAnalysis)

## Задача
Сервер T-Invest считает технические индикаторы (SMA/EMA/RSI/MACD/BB) у себя
(`MarketDataService/GetTechAnalysis`). Цель — из тех же закрытий получить **те же числа
до последнего печатаемого знака**, не подсматривая эталонную реализацию. Сложность: конвенции
расчёта нигде не задокументированы, их надо вывести из пар вход→выход. Оракул беспощадный:
сервер отдаёт число, и оно либо совпадает, либо нет.

## Оракул
Серверные значения на тестовом окне (2026-01-01 … 2026-06-19, дневной интервал, цена закрытия).
Критерий «бит-в-бит»: после округления моей реконструкции и значения сервера до 6 знаков —
равны (6 знаков = точность, с которой API печатает Quotation).

## Итог
**2463 / 2464 бар-значений совпали бит-в-бит** (16 рядов × 154 бара × 2 инструмента).
Общий maxΔ = 5.13e-7. Единственный промах — **ничья округления**, не ошибка конвенции (см. ниже).

| инструмент | параметры | результат |
|---|---|---|
| SBER | SMA20 EMA20 RSI14 MACD(12,26,9) BB20/k=2.0 | 1232/1232 бит-в-бит |
| GAZP | SMA50 EMA50 RSI21 MACD(8,21,5) BB20/k=2.5 | 1231/1232 (1 ничья округл.) |

Второй инструмент с другими длинами и **дробным** k=2.5 — защита от переобучения под SBER/дефолты.

## Вскрытые конвенции (главное)
1. **SMA / BB-middle** — простое среднее последних N закрытий. Совпадение до 1e-13 (чистая
   арифметика, без рекурсии). Сервер использует историю ДО запрошенного окна — индикатор отдаёт
   значение уже на первом баре окна.
2. **EMA** — α = 2/(N+1), рекурсия. **Seed-инвариантна при прогреве:** обе гипотезы посева
   (seed = SMA первых N · seed = первое закрытие) дали *идентичную* ошибку 4.99e-7. Это прямое
   подтверждение, что влияние стартовой точки затухает как (1−α)ⁿ: на ~1600 барах прогрева
   разница ниже 6-го знака. То есть «правильного» seed знать не нужно — достаточно длинной истории.
3. **RSI** — сглаживание **Уайлдера** (avg = (prev·(N−1)+cur)/N), а не SMA-сглаживание. Гипотеза
   проверена: Wilder совпал бит-в-бит, на обеих длинах (14 и 21).
4. **MACD** — линия = EMA(fast) − EMA(slow); сигнал = EMA(signal) от линии MACD. Совпало и на
   (12,26,9), и на (8,21,5).
5. **Bollinger Bands** — **дисперсия популяционная (÷N)**, не выборочная (÷N−1). Дискриминирующий
   результат: гипотеза «sample» отвалилась с maxΔ ≈ 0.2–0.3 на ВСЕХ барах, population — 5e-7.

## Единственный промах и почему это не ошибка
GAZP EMA(50) @ 2026-01-30: recon = 125.320074, oracle = 125.320075 — расхождение ровно на **одну
единицу последнего печатаемого знака** (Δ = 1e-6, на 1 баре из 154). Полноточное значение легло
на границу .5 шестого знака; float64 при ином (алгебраически эквивалентном) порядке операций
округлил в другую сторону. Это **ничья округления**, качественно отличная от ошибки конвенции:
ошибка конвенции (как BB-sample выше) даёт Δ ≈ 0.2 на ВСЕХ барах, а не 1e-6 на одном.
Порог НЕ задирался ради «16/16» — промах показан как есть и классифицирован.

## Практический вывод для проекта
Индикаторы T-Invest полностью воспроизводимы локально из свечей по этим конвенциям. Это значит:
их можно считать офлайн (без расхода лимита 600/мин на каждый индикатор), строить на любых
параметрах/таймфреймах и использовать в бэктестере (`backtest/`) с гарантией, что значения
совпадут с тем, что увидит трейдер в приложении. Ключевое требование — прогрев историей
до начала окна (для EMA/RSI/MACD ≥ нескольких сотен баров, для SMA/BB ровно N−1).
