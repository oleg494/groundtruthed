# Калибровка ATR-фильтра daybot — результат

**Скрипт:** `analysis/atr_filter_calibration.py` · **Прогон:** 2026-07-02 ·
**Данные:** реальные свечи GetCandles (sandbox-домен, read-only, кэш `backtest/.cache`)

## Что проверяли

daybot (`daybot/run.py`) пропускает день, если ширина утреннего диапазона 10:00–10:30 МСК
не попадает в коридор `[ATR_MULT_MIN, ATR_MULT_MAX] × ATR(100)` = **[0.6, 2.0]**.
Гипотеза аудита: утренняя 30-мин свеча почти никогда не даёт ≥60% дневного хода —
фильтр отрежет практически все дни, бот не будет торговать.

Метрика: `ratio = (high−low утренней 30-мин свечи 10:00 МСК) / ATR(100)` по дневным барам,
ATR — та же формула, что в `daybot.get_atr` (среднее TR за 100 дней), считается строго по
дням ДО текущего (без lookahead). Ratio безразмерный — пункты фьючерсов сокращаются.

## Вердикт: фильтр ДУШИТ

Гипотеза **подтверждена** по всем трём инструментам:

- Медиана ratio — **0.12–0.24**, т.е. типичная утренняя 30-минутка покрывает лишь
  12–24% дневного ATR. Порог 0.6 лежит выше p90 у всех троих.
- Текущий коридор пропускает **1.5–5.5% дней** — 1–2 сделки в пару месяцев на
  инструмент. Форвард-тест с таким фильтром статистики не наберёт никогда.
- Верхняя граница 2.0 — почти мёртвая: p90 ≤ 0.46 у всех, максимумы ratio —
  0.816 (BMQ6), 0.915 (NGN6), 2.661 (GLDRUBF); за всю историю порог 2.0 срезал
  ровно один день (GLDRUBF). Реально работает только нижняя граница.

Причина ошибки калибровки очевидна: порог 0.6 осмыслен для сравнения диапазона
с ATR **того же таймфрейма**, а тут 30-мин окно сравнивается с **дневным** ATR —
разные масштабы (√t: 30 мин ≈ 1/28 торгового дня → ожидаемая доля хода ~0.2).

## Рекомендация порогов

- **`ATR_MULT_MIN = 0.1`** — общий для всех: BMQ6 71.9%, NGN6 60.6%, GLDRUBF 94.5%
  проходящих дней. Отрезает только совсем «мёртвые» утра (нижние ~10–40% распределения),
  оставляя боту материал для статистики.
- Точнее — **по инструментам**: BMQ6/NGN6 `0.1`, GLDRUBF `0.15–0.2` (у золота
  распределение сдвинуто вправо, при 0.2 остаётся 61.2% — фильтр начинает
  реально отбраковывать тихие дни, а не всё подряд).
- **`ATR_MULT_MAX`**: снизить с 2.0 до **~0.5** — тогда верхняя граница начнёт ловить
  аномально широкие утра (у GLDRUBF p90=0.46, т.е. отрежет ~10% самых бурных дней,
  когда ход уже сделан до пробоя). При 2.0 параметр — плацебо.

Оговорка честности: у BMQ6/NGN6 всего 64–66 дней с прогретым ATR(100) — контракты
молодые, дневной истории мало (первые ~100 дней уходят на прогрев). Квантили по ним
менее устойчивы, чем у GLDRUBF (165 дней), но вывод «0.6 выше p90» от этого не
меняется — запас порядковый, не маргинальный.
